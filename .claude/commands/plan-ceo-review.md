# /plan-ceo-review — 창업자 관점 제품 전략 리뷰

코딩 시작 전 방향성과 범위를 검토하는 스킬.
**항상 코드 작성 전에** 실행. 방향이 틀리면 완벽한 코드도 쓰레기.

## 트리거 조건
- "이게 맞나?", "더 크게 생각해줘", "전략 리뷰해줘"
- "방향성 검토", "CEO 리뷰"
- 새 기능 기획 시작 전, 큰 리팩터링 전

---

## STEP 1: PRE-REVIEW 시스템 감사

리뷰 시작 전 현재 상태 파악 (읽기 전용).

```bash
git log --oneline -20
git diff main --stat 2>/dev/null || git diff HEAD --stat
```

다음 파일도 읽기:
- `CLAUDE.md` — 현재 규칙 + 스킬 목록
- `memory/project_status.md` — 진행 중인 작업 현황
- `memory/project_deferred_tasks.md` — 보류 중인 작업

**감사 질문:**
- 지금 진행 중인 작업과 충돌하는가?
- 보류 작업과 번들할 수 있는가?
- 이미 완료된 것을 다시 하려는 건 아닌가?

## STEP 2: 스코프 모드 선택

4가지 모드 중 하나를 AskUserQuestion으로 물어볼 것:

| 모드 | 설명 |
|------|------|
| **10x 확장** | 10배 더 좋은 제품을 꿈꾼다. 모든 확장 기회 제안. 각각 opt-in |
| **선택적 확장** | 기본 범위 유지 + 가치 높은 확장만 선별 제안 |
| **범위 유지** | 최대 엄격성. 현재 계획을 bulletproof하게만 |
| **MVP 축소** | 핵심 목표 달성하는 최소 버전. 나머지 전부 제거 |

모드를 모르겠으면 선택적 확장을 기본으로.

## STEP 3: 우나어 특화 CEO 체크리스트

각 항목 Yes/No로 판단 후 이유 1줄:

### 사용자 가치
- [ ] 50대+ 사용자 입장에서 진짜 문제를 해결하는가?
- [ ] "우리 또래" 브랜드 목소리와 일치하는가? ("시니어", "노인" 금지)
- [ ] 사용자가 이것 때문에 다시 오는가? (리텐션 기여)

### 지표 기여
- [ ] DAU / 공감수 / 댓글수 중 어떤 지표에 기여하는가?
- [ ] 측정 가능한 성공 기준이 있는가?

### 시스템 연계
- [ ] AI 에이전트 시스템 (CEO, CMO, CTO 등)과 연계가 있는가?
- [ ] 어드민 패널 영향은 없는가? (있으면 어드민도 같이 수정 계획)
- [ ] Band API 심사 완료 후 연계할 부분이 있는가?

### 완전성 (Boil the Lake)
- [ ] 완전한 구현인가, 지름길인가?
- [ ] AI가 있으면 완전한 버전과 지름길 비용 차이가 거의 없음. 완전한 버전 선택

## STEP 4: 8가지 Prime Directives 점검

각 항목을 계획에 적용:

1. **Zero silent failures** — 실패 시 Slack 알림 / BotLog 기록 계획이 있는가?
2. **Every error has a name** — 구체적 에러 클래스/시나리오가 명시됐는가? (`AppError`, `NotFoundError`, `ForbiddenError`)
3. **Data flows have shadow paths** — 정상 경로 외 null/empty/upstream 에러 경로도 설계했는가?
4. **User interactions have edge cases** — 빠른 클릭, 뒤로가기, 느린 연결, 만료된 세션 처리?
5. **Observability is scope** — 모니터링은 afterthought이 아닌 필수 구현 항목으로 포함됐는가?
6. **Diagrams are mandatory** — 복잡한 데이터 흐름이면 ASCII 다이어그램 포함할 것
7. **Everything deferred must be written** — 보류 결정은 `memory/project_deferred_tasks.md`에 기록
8. **Optimize for 6 months** — 오늘 문제 해결 + 3개월 후 이 코드가 악몽이 되지 않는가?

---

## STEP 5: CEO 사고 패턴 적용

계획에 대해 아래 렌즈로 질문:

- **집중 = 제거**: 무엇을 안 할지가 가치. 지금 제거할 수 있는 것은?
- **역전 반사**: "어떻게 이 기능이 실패할까?" — 실패 시나리오 3가지
- **속도 보정**: 돌이킬 수 없는 결정인가? 그렇다면 느리게. 아니면 빠르게 진행
- **디자인 = 신뢰**: 이 UI 변경이 50대+ 사용자의 신뢰를 쌓는가, 깎는가?
- **엣지 케이스 편집증**: 닉네임이 20자이면? 결과가 0건이면? 네트워크 오류면?
