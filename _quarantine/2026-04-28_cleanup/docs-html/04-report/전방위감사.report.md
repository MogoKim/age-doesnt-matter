### FIX-11: 인메모리 레이트 리밋 분산 환경 무효화 (Critical)

#### 문제 분석
```
문제 1: Vercel 다중 인스턴스 → 각 인스턴스 독립 Map 메모리
  - Rate Limit: 60요청/분 설정
  - 실제 동작: 인스턴스1 60 + 인스턴스2 60 + ... → 총 180~300 요청 통과 가능
  - 결과: 레이트 리밋이 제 역할 하지 못함

문제 2: IP 스푸핑 취약점
  - x-forwarded-for 첫 번째만 추출 → split[0] 스푸핑 가능
  - 예: "attacker.com, real-ip" → attacker.com으로 인식

문제 3: 폴백 로직 결함
  - unknown 폴백 → 전체 사용자가 동일 "unknown" 키로 공유
  - 결과: 한 사용자의 DDoS가 모든 사용자 차단
```

#### 수정 내용

**src/lib/rate-limit.ts** (신규 함수 추가):

```typescript
import { Ratelimit } from "@upstash/ratelimit";
import { Redis } from "@upstash/redis";

const redis = new Redis({
  url: process.env.UPSTASH_REDIS_REST_URL!,
  token: process.env.UPSTASH_REDIS_REST_TOKEN!,
});

const ratelimit = new Ratelimit({
  redis: redis,
  limiter: Ratelimit.slidingWindow(60, "1 m"),
  analytics: true,
});

export async function checkRateLimit(identifier: string) {
  const { success } = await ratelimit.limit(identifier);
  return success;
}
```

**src/lib/api-rate-limit.ts** (신규 파일):
```typescript
export async function checkApiRateLimit(req: Request): Promise<boolean> {
  const ip = extractClientIp(req);
  try {
    return await checkRateLimit(ip);
  } catch (error) {
    // Upstash 장애 시 무중단 폴백 (in-memory)
    return fallbackRateLimit(ip);
  }
}

function extractClientIp(req: Request): string {
  // 우선순위: x-real-ip → x-forwarded-for(마지막) → fallback
  const realIp = req.headers.get("x-real-ip");
  if (realIp && realIp !== "unknown") return realIp;

  const forwarded = req.headers.get("x-forwarded-for");
  if (forwarded) {
    const ips = forwarded.split(",").map(ip => ip.trim());
    return ips[ips.length - 1]; // 마지막 IP (신뢰할 수 있는 프록시)
  }

  return "anonymous";
}
```


```typescript
const pool = new Pool({
  connectionString: DATABASE_URL,

    rejectUnauthorized: false,     // Supabase PgBouncer 자가서명 인증서 수용
```

### FIX-16: Threads 토큰 만료 추적 미흡 (High)

#### 문제 분석
```
agents/cmo/platforms/threads-token-refresh.ts에서:

1. refreshLongLivedToken()이 token만 반환
   → expires_in 정보 폐기 → BotLog에 저장 불가
   
2. 토큰 만료 시점을 파악할 수 없음
   → Slack 선제 경고 불가능
   → 장애 발생 후 대응 (reactive)

3. 토큰 갱신 이력이 BotLog에 기록되지 않음
   → 운영자가 갱신 여부 확인 불가
```

#### 수정 내용
**agents/cmo/platforms/threads-client.ts**:
```typescript
// 변경 전
export async function refreshLongLivedToken(token: string): Promise<string>

// 변경 후
export async function refreshLongLivedToken(
  token: string
): Promise<{ token: string; expiresIn: number }> {
  const response = await fetch(
    `https://graph.instagram.com/refresh_access_token?grant_type=ig_refresh_token&access_token=${token}`
  );
  const data = await response.json();
  return {
    token: data.access_token,
    expiresIn: data.expires_in, // 초 단위
  };
}
```

**agents/cmo/platforms/threads-token-refresh.ts**:
```typescript
const { token, expiresIn } = await refreshLongLivedToken(currentToken);

const expiresAt = new Date(Date.now() + expiresIn * 1000);
const daysUntilExpiry = Math.floor(
  (expiresAt.getTime() - Date.now()) / (24 * 60 * 60 * 1000)
);

// BotLog 저장
await db.botLog.create({
  data: {
    agentType: "CMO",
    action: "THREADS_TOKEN_REFRESH",
    status: "SUCCESS",
    details: {
      expiresAt: expiresAt.toISOString(),
      daysUntilExpiry: daysUntilExpiry,
    },
  },

// 7일 전 선제 경고
if (daysUntilExpiry <= 7) {
  await notifySlack({
    channel: "#알림-긴급",
    blocks: [
      {
        type: "section",
        text: {
          type: "mrkdwn",
          text: `⚠️ *Threads 토큰 만료 예정*\n만료까지: *${daysUntilExpiry}일*\n예상 시각: ${expiresAt.toLocaleString("ko-KR")}`,
        },
      },
    ],
  });
}
```
