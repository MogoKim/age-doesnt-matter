## 2. 성능 분석

### 2.1 잘 되어 있는 것 ✅
- **이미지 최적화**: next/image 사용, `/_next/image` 쿼리로 크기 최적화 (w=256, w=384)
- **폰트 로딩**: WOFF2 + `display: swap` → FOIT 방지
- **코드 스플리팅**: 페이지별 JS 청크 분리 (layout.js, page.js 별도)
- **동적 임포트**: TipTap 에디터 `dynamic()` 적용
- **병렬 데이터 로딩**: 홈 `Promise.all()` 적용
- **스켈레톤 UI**: 주요 페이지 6개 loading.tsx 구현
- **스크롤 스냅**: 일자리 카드 가로 스크롤 최적화

### 2.2 개선 필요 ⚠️

| 이슈 | 위치 | 심각도 | 설명 |
|------|------|--------|------|
| `sizes` 속성 미지정 | 다수 Image 컴포넌트 | 중간 | 반응형 이미지 최적화 미흡, 불필요한 큰 이미지 다운로드 가능 |
| `use client` 과다 | GNB.tsx, IconMenu.tsx | 낮음 | pathname만 필요한데 전체 컴포넌트가 클라이언트 번들에 포함 |
| useMemo 미적용 | CommentSection.tsx:29 | 중간 | `sortComments()` 매 렌더마다 재실행 |
| useCallback 미적용 | FAB.tsx:49-51 | 낮음 | `goTo` 함수 매번 재생성 |
