3. 실험 후보 3개 (데이터가 정한 우선순위)

🥇 앱-A · 더 둘러보게 (화면수↑) 1순위

💡 가설 첫날 화면수가 D1 최강 동인(1→6화면 = 3%→17%). 앱에서 다음 글·관련 콘텐츠 동선을 강화하면 화면수↑ → D1↑.
  • 이미 진행 중: 웹 실험 1(본문 직후 "다음에 읽기 좋은 이야기" 카드)은 앱에도 적용되고 있습니다(TWA 제외 안 함). → 먼저 실험 1의 앱 세그먼트 결과를 봐서 앱에서도 화면수가 느는지 확인.
  • 추가 안: 앱 홈/글 하단에 "다음 글 이어보기" 동선 강화, 베스트·커뮤니티 추천(재방문 견인 콘텐츠).

난이도 낮음(웹 실험 자산 재사용). 표본 충분(1,200). 가장 확실.

🥈 앱-B · 첫 참여 유도 (댓글·공감) 2순위

💡 가설 참여한 사람 D1 35.7%(안 한 사람 4.7배). 단 참여율이 낮음(1.2%). 첫 댓글/공감 한 번을 쉽게 만들면 D1↑.
  • 예: 글 하단에 "한 줄 공감" 빠른 버튼, 댓글 입력 허들 낮추기, 첫 댓글 유도 문구.
  • 주의: 참여 표본(14명)이 작아 "참여→D1" 인과는 아직 약함. 실험으로 검증 필요(참여를 늘렸을 때 D1이 정말 오르나).

난이도 중. 효과 잠재력 큼(5배 신호)이나 검증 필요.

🥉 앱-C · 푸시 구독률↑ → 이후 리인게이지 3순위(2단계)

💡 가설 현재 구독률 1.6%. 1단계: 앱 회원 푸시 구독 유도 강화(타이밍·문구) → 구독률↑. 2단계: 구독자에게 다음날 "새 인기글/내 글 반응" 리인게이지 푸시 → D1↑.
  • 구독률이 의미 있는 수준(예: 10%+)으로 오른 뒤에야 리인게이지 푸시가 효과를 냅니다.
  • 비용·정책: 리인게이지 푸시는 마케팅 동의·야간(21~08) 차단 정책 준수 필요.

난이도 중~상(구독 유도 + 자동발송 신설). 지금 당장은 효과 미미 → 둘러보기·참여 다음.
