## 1. Executive Summary — 3대 발견

### 발견 ① 모든 회원 지표가 봇으로 오염돼 있었다
User 231명 중 **진짜 카카오 가입자는 46명뿐**. 나머지는 curator 봇 95 + seed봇 83 + seed 7 = 185개.
"광고로 주 74명 가입"은 대부분 curator 봇이었다. **진짜 가입은 주 3~14명.**
→ 어드민 "총 회원수"·CDO KPI는 봇을 포함해 부풀려져 있다 (§6).

### 발견 ② 온보딩 닉네임 검증 버그가 가입자를 가둔다
온보딩 미완 이탈 유저 5명이 **전원 `user_XXXXX` 자동생성 닉네임**.
`checkNickname`의 정규식 `/^[가-힣a-zA-Z0-9]+$/`이 **언더스코어를 금지**하는데,
자동 닉네임은 `user_24730196`처럼 `_`를 포함 → 그대로 "다음" 누르면 영원히 step1 통과 불가.
→ 재방문 의지가 있는 user_24730196은 6/1·6/3 양일 반복 시도했지만 매번 막혔다 (§7).

### 발견 ③ "가입 전 글을 정독한 사람"은 전원 재방문했다
가입 전 콘텐츠를 정독(`post_view`)한 4명 → **재방문 100%**.
정독 안 한 42명 → 재방문 14%. 활동(글/댓글) 쓴 유저의 재방문율도 압도적.
→ **콘텐츠로 마음이 동해서 가입한 사람이 남는다.** 가입을 서두르게 하는 것보다,
   가입 전에 좋은 글을 읽게 만드는 것이 리텐션의 핵심.

### 한 줄 처방
> **(즉시) 온보딩 닉네임 버그를 고쳐 가입 자체를 막는 누수를 멈추고,**
> **(구조) "가입 전 정독 → 가입 → 첫 활동" 동선을 강화한다.** 봇 지표부터 걷어내야 진실이 보인다.

## 7. 온보딩 닉네임 버그 (코드 근거)

> ⚠️ **정정 (2026-06-07 재검증 — 이 절의 결론은 틀렸음)**
> 전체 파이프라인을 코드로 역추적한 결과 **이 버그는 존재하지 않는다.**
> - `onboarding/page.tsx`는 `OnboardingForm`에 `callbackUrl`만 전달하고, `OnboardingForm`의 닉네임 input은 `useState('')` — **항상 빈칸으로 시작**한다(git 최초 구현부터 동일).
> - 자동닉네임 `user_XXX`(`auth.ts:87`)는 **DB·JWT의 `needsOnboarding` 판정에만** 쓰이고, **온보딩 input에 채워지는 경로가 없다.**
> - 따라서 정규식의 `_` 금지는 자동닉네임을 막지 않는다(아래 §7 단계 2가 코드와 불일치 = 분석가의 코드 오독).
> - **단** user_24730196·user_25815581의 온보딩 이탈은 실측이며, 진짜 원인은 정규식이 아니라 **"빈칸 + 새 닉네임 강제 입력"이 50–60대에게 막막한 UX 허들**일 가능성. → 처방은 "정규식 수정"이 아니라 **온보딩 닉네임 단계 안내 강화**로 재정의.

**파일**: `src/lib/actions/onboarding.ts:11,31`

```typescript
const NICKNAME_REGEX = /^[가-힣a-zA-Z0-9]+$/   // ← 언더스코어(_) 불허
// checkNickname: 정규식 실패 시 "한글, 영문, 숫자만 사용할 수 있어요"
```

**문제 메커니즘**:
1. 카카오 가입 시 닉네임이 `user_24730196`처럼 **언더스코어 포함**으로 자동 생성
2. 온보딩 step1에서 그 값이 입력칸 기본값 → 사용자가 그대로 "다음" 클릭
3. 정규식이 `_`를 거부 → `nicknameStatus !== 'valid'` → step1 통과 불가 (`OnboardingForm.tsx:143`)
4. 50~60대가 "새로 입력해야 한다"를 모르면 영원히 갇힘 → 이탈

**증거**: 온보딩 미완 이탈 5명 전원이 `user_XXXXX` 닉네임 (행동 흔적상 step1에서 멈춤).

**부수 문제**: `signup_abandoned`는 `beforeunload`로 발생하는데 페이지 이동·리다이렉트마다
오발생 → 이벤트가 과다 집계(측정 노이즈). 실제 이탈 수보다 부풀려짐.

**처방**: ① 닉네임 정규식에서 자동닉네임 통과 처리 또는 기본값을 빈칸으로 + 안내문구
② signup_abandoned를 실제 이탈에만 발생하도록 정비.
