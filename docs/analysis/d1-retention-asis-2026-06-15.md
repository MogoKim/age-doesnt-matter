# D0→D1 리텐션 인과 분석 (기간: 2026-06-01 ~, 비회원 성숙 2,986세션)

> 목적: "내일(D1) 다시 오게"의 동인을 **인과/상관 구분**까지 규명. 봇 제외, 코호트 보정.
> 정의: page_view=화면 이동 수(목록 포함) / post_read=글 상세(커뮤니티·일자리) 실제 조회. ⚠️ 매거진 상세엔 post_read 미측정.

## ⚠️ 이전 분석 정정
이전에 "글 클릭(post_read)이 D1 동인"이라 했으나 **틀렸음**. 데이터는 **"사이트를 둘러봤나(화면 수)"가 동인**이고 글 클릭 자체는 약함을 보여줌(아래 Q1).

---

## Q1. 동인 = "글 클릭"이 아니라 "둘러봤나(화면 수)"

| 본 글 수(post_read) | D1 | | 본 화면 수(page_view) | D1 |
|---|---|---|---|---|
| 글 0개(목록만) | 4.8% | | 1화면 | 2.1% |
| 글 1개 | 2.4% | | 2화면 | **11.8%** |
| 글 2개 | 5.5% | | 3~4화면 | 13.3% |
| 글 3개+ | 13.4% | | 5화면+ | 21.2% |

**화면 통제 검증(둘 다 화면 2+):** 글0개 15.6% vs 글1개+ 19.4% → **차이 작음.**
→ **D1 동인은 "둘러봤나(2화면 이상)". 글 하나 클릭은 약함**(글1개 2.4% < 목록만 4.8%, 단발 글소비는 오히려 낮음). 글3개+(13.4%)는 "깊은 탐색"의 결과.
→ **매직넘버 = 2번째 화면**(1→2화면 2.1%→11.8%, 5.6배).

## Q2. 목록 정의 + 어디서 멈추나 (첫 진입별 정확 숫자)
목록 = 글 상세가 아닌 페이지(홈·게시판목록·매거진목록·일자리목록·베스트).

| 첫 진입 | 세션 | 글로 넘어간 비율 | D1 |
|---|---|---|---|
| 홈 | 1,394 | 9.6% | 8.5% |
| 글상세 단발진입(검색 등) | 808 | 56% | **1.9%** |
| 매거진목록 | 232 | **0%** | 0.4% |
| 게시판목록 | 223 | 4.9% | 4.5% |
| 일자리목록 | 180 | 1.1% | **0%** |
| 베스트 | 65 | 15.4% | **12.3%** |

→ **홈 진입자(최다)의 90%가 글 미클릭.** 검색 등 글 하나만 보고 들어온 808명이 D1 1.9%(최악). 매거진·일자리 목록은 글 클릭 0~1%(막다른 길).

## Q3. 베스트 = 인과 (상관 아님)
같은 "본 글 수" 구간 내 비교(행동 통제):
| 구간 | 베스트 봄 | 안봄 |
|---|---|---|
| 글 0개 | **15.7%** | 3.8% |
| 글 1개+ | **26.3%** | 4.1% |
→ 글 수를 맞춰도 베스트 본 사람 4~6배. **베스트는 D1을 만드는 인과적 효과.**

## Q4. 앱 = 구조적 동인 (선택편향만 아님)
같은 "본 글 수" 구간 내 앱 vs 웹:
| 구간 | 앱 | 웹 |
|---|---|---|
| 글 0개(목록만) | **10.4%** | 1.2% |
| 글 1개+ | **21.7%** | 1.6% |
→ 행동 통제해도 앱 8~13배. **앱 자체(홈 아이콘·재방문 용이)가 구조적 D1 동인.**

## 🎯 종합 (인과 판정)
1. **동인 = 둘러보게(2화면+) 만들기.** 글 클릭 아님. 매직넘버 = 2번째 화면.
2. **누수: 홈 진입 90% 글 미클릭 + 검색 단발진입 808명 D1 1.9%.**
3. **베스트 = 인과적 D1 4~6배** → 노출 강화 가치.
4. **앱 = 구조적 동인 8~13배** → 설치가 리텐션 인프라.
