## 한 줄 결론

**매거진만의 문제가 아니다. 우나어 전체가 더딘 진짜 원인은 "도메인 권위(외부 백링크)가 거의 0"이라서다.**
색인·기술 설정은 이미 잘 돼 있다. 구글은 우리 3,000여 페이지를 *발견*했지만, 신뢰(권위) 신호가 없어 *색인·순위 부여를 보류*하고 있다. 그래서 경쟁 없는 **브랜드 검색만** 1페이지에 뜨고, 나머지는 전부 2~3페이지에 묶여 클릭이 0에 가깝다.

이건 버그가 아니라 **저권위 신규 도메인의 정상적인 "샌드박스" 국면**이다. 그리고 정체가 아니라, 6월 들어 클릭이 급증하며 **가속이 막 시작**됐다.

## 1. 먼저 — 문제가 *아닌* 것 (창업자 지적에 동의)

온페이지·기술 SEO는 코드 확인 결과 **정상**이다. 여기를 더 손봐도 노출은 안 는다.

| 항목 | 상태 | 근거 |
|---|---|---|
| robots 색인 허용 | 정상 (`index:true, follow:true`) | `src/app/layout.tsx:44` |
| GSC·네이버 사이트 인증 | 정상 (메타 존재) | `src/app/layout.tsx:48-53` |
| 표준 URL(canonical)·OG·구조화데이터 | 정상 (Article/FAQ/Breadcrumb JSON-LD) | `src/app/(main)/magazine/[id]/page.tsx:65-218` |
| 사이트맵 | 정상 — GSC "성공", 3,152~3,230 페이지 발견 | `src/app/sitemap.ts` |
| www→non-www 통합 | 정상 (301 영구 리다이렉트) | `next.config.js:58` |
| 구글 색인 API | 작동 (매거진 발행 시 자동 요청) | `agents/cafe/indexing-api.ts` |

→ **설정/색인은 병목이 아니다.** 병목은 사이트 바깥의 신뢰(권위)와 콘텐츠 신호다.

## 3. 근본 원인 (전사, 우선순위)

1. **외부 권위(백링크) 부재 — 전사 최대 원인.**
   백링크는 구글이 "이 사이트를 믿어도 되나"를 판단하는 핵심 신호다. 5개(그것도 에디토리얼 0)로는 **어떤 주제도 1페이지에 못 올린다.** 그래서 매거진만이 아니라 커뮤니티·잡·전 영역이 똑같이 2~3페이지에 묶인다. 유일한 예외가 브랜드 키워드(경쟁이 없어서).

2. **볼륨↔품질 불균형.**
   권위 없이 페이지 수만 3,000+. 얇거나 유사한 페이지(카페 재작성·잡 리스트·짧은 AI 매거진)가 사이트 평균 품질 신호를 희석해, 구글이 색인을 더 보수적으로 배급한다. 지금은 **양보다 집중**이 필요한 국면이다.

3. **콘텐츠↔검색쿼리 정렬 약함.**
   매거진 제목이 에세이형("내 몸이 안 이뤄짐", "50대 일한다는 것의 의미")이라 *아무도 검색하지 않는 쿼리*를 타겟한다. 롱테일 키워드 사전(`agents/magazine/longtail-keywords.ts`, 79개)은 존재하나 "자연스럽게 녹여쓰기"라는 약한 힌트로만 쓰여(`agents/magazine/prompt.ts:97`), "1글 = 1개 정확매칭 쿼리" 규율이 없다.

4. **내부링크가 네비게이션 위주.**
   47,124개 중 다수가 메뉴 링크다. 콘텐츠 간 주제별(contextual) 링크가 적어 권위가 딥 콘텐츠로 흐르지 않는다.

5. **(경미) GSC에 www/non-www 사이트맵 이중 등록.**
   301로 대체로 무해하나, 레거시 www 피드(2023 아임웹 잔재 포함)는 정리 가능하다.

## 4. 통합 성장 로드맵 (A: 권위 + B: 콘텐츠 집중 + C: 브랜드)

**전략 논리:** DA≈0 + 3천 얇은 페이지 상태에서 "오가닉 폭발"의 최단 경로 =
① 희석 중단(품질 집중) → ② DA 무관 초롱테일 선점(즉시 1페이지) → ③ 권위 빌딩(경쟁 키워드 해금, 수개월) → ④ 브랜드 증폭(플라이휠).

### Phase 0 — 기준선 + 정리 (즉시, ~1주)
- GSC **페이지(색인생성) 리포트**에서 "발견됨–색인 안 됨 / 크롤링됨–색인 안 됨" 사유별 카운트 확보 → 원인②를 정량 확정 + 측정 기준선.
- 얇은/저가치 페이지(자동 잡 리스트·짧은 매거진·유사 커뮤니티) 식별·목록화.

### Phase 1 — 콘텐츠 집중(B) + 퀵윈 (2~4주) · 주로 코드/콘텐츠
- **thin 페이지 prune/noindex**: 사이트맵에서 제외하고 `PostStatus`로 정리 → 사이트 품질 신호 집중.
- **퀵윈**: 노출 있는 page-2(5~15위) 글의 제목·H1·첫 문단을 **검색 쿼리 정확매칭**으로 보강 (예: 50대-이력서 13.7위, 주부도-첫투자 7.8위).
- **매거진 키워드-퍼스트 전환**: 주제 선택을 트렌드 제안 → *미사용 정확매칭 롱테일 쿼리*로 역전. 제목 = 그 질문형 쿼리. **1글 = 1쿼리.** (진입점: `agents/cafe/magazine-generator.ts` 주제선택부 461-499, `agents/magazine/prompt.ts` 제목규칙.)
- **토픽 클러스터**: `pillar` 필드 기준 필라 페이지 생성 + **본문 contextual 내부링크**(네비 링크 말고).

### Phase 2 — 권위 빌딩(A) (지속, 1~3개월) · 주로 마케팅(코드 아님)
- 5060 관련 매체 기고·보도자료·제휴(복지관·평생교육원·지역 커뮤니티).
- 자체 통계/설문 콘텐츠로 citation(인용 링크) 유도.
- 디렉토리를 넘어 **에디토리얼 백링크** 확보 — referring domains 수를 늘리는 게 핵심.

### Phase 3 — 브랜드 플라이휠(C) (지속)
- 이미 작동하는 브랜드 검색(63%)을 증폭: 운영 중 SNS(Threads/IG/FB/X)·카페 노출로 브랜드 인지를 키운다.
- branded search↑ → 구글 권위 신호↑ → 비브랜드 순위 상승의 선순환.

## 6. 다음 액션 (창업자 의사결정)

이 보고서는 **진단까지**다. 로드맵 각 Phase의 실제 실행은 Phase별로 착수 결정한다.
- 가장 빠른 자체 실행 가능 항목 = **Phase 0(기준선) + Phase 1(콘텐츠 집중/퀵윈)** — 코드/콘텐츠 작업이라 내부에서 바로 가능.
- 가장 큰 레버 = **Phase 2(권위 빌딩)** — 마케팅 실행이 필요하며 효과는 수개월에 걸쳐 누적.
