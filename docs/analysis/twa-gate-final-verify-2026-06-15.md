## ⚠️ 먼저 — 제 지난 분석 정정 (중요)

**지난번 "배정 분모 ~2배(2.07) 부풀림" 주장은 과장이었습니다. 직접 측정하니 ~1.00배(부풀림 거의 없음)입니다.**

- 지난번 2.07 = "가입자 userId에 연결된 **전체 이벤트** sessionId 수"였다. 이건 가입자가 OAuth로 앱→외부→복귀하며 만든 세션까지 포함 → 부풀어 보였다.
- 이번 직접 측정 = "**배정 이벤트(twa_gate_assigned)** 세션이 몇 명으로 수렴하나": userId로 확인되는 배정 세션 27개 → 실제 27명 = **1.00배**.
- 즉 **배정 이벤트 자체는 사람당 1개**에 가깝다(sessionStorage 세션당 1회 가드 + _anon_sid 30일 쿠키라 잘 안 바뀜).

→ R1(sessionId 부풀림)의 실제 심각도는 내가 말한 것보다 **훨씬 작다**. 정직하게 정정한다.

## 내가 못 보는 한계 (투명 공개)
- `_uid`(디바이스ID)는 클라이언트 localStorage 전용 → 서버 EventLog에 없음. 미가입 배정세션 212개의 진짜 사람 수는 직접 못 셈(부풀림 상한 미상, 단 가입자분으로 보면 1배).
- GA4(gtag)는 어드민과 별개 소스 → 대조 불가하나 어드민 정합성엔 무관.
- 표본 작음(가입 71). 배정 6/12 이전은 소급 불가.
- userId 붙은 게이트 이벤트(assigned 27·view 43 등) = 로그인 상태 발생. 소수 노이즈, 집계 영향 경미.
