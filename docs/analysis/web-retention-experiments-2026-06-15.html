디멘드젠 웹 유입, D1 리텐션을 어떻게 올릴까

광고를 디멘드젠(여성 45~70 · 웹 랜딩=베스트)으로 전환합니다. 웹 유입을 다음 날 다시 오게(D1) 만들기 위한 실험 제안서.
✔ 이 버전은 현재 화면(AS-IS)을 코드 8개 파일과 1:1 대조해 확정했습니다

📋 이 문서 30초 요약
  • 문제: 웹 유입 D1 1~3% (앱 12%의 1/10).
  • 아는 정답: 첫날 3화면 이상 둘러보면 D1 12.7%. → "한 글만 보고 나가지 않게".
  • 방향: 정답을 미리 정하지 않고 실험으로 비교. 실험은 2개.
  • 이번 v3 차이: 과거 관리문서 대신 코드를 직접 읽어 현재 화면(대조군)을 확정했습니다 — 실험군을 정확히 나누려면 대조군이 정확해야 하므로.
⚠️ 코드는 한 줄도 안 바꿨습니다. 이건 제안서입니다. 실험 방식을 정해주시면 그때 개발합니다.

2. 실험 1 · "다음 글" 동선 ⭐ 1순위

💡 가설 글을 다 읽은 직후 "다음 글"이 바로 보이면 한 글 더 클릭 → 3화면 도달 ↑ → D1 ↑. 현재는 관련글 12개가 맨 아래(광고·댓글·가입CTA 뒤)에만 있어 대부분 도달 전에 이탈.
🧭 딥다이브 — 두 가지를 절대 안 건드린다: 광고 수익 · 댓글
글 상세 광고는 3개이고 위치가 이렇습니다(코드 확인):
본문 → [광고① 인아티클·애드센스] → 댓글 → 가입CTA → [광고② 쿠팡] → 관련글 12개 → [광고③ 하단·애드센스] 광고 노출 = 사용자가 그 자리까지 스크롤해야 발생합니다. 그래서 다음 글 카드를 광고① "다음"에 둡니다 → 가장 비싼 광고①은 본문 직후 그대로 노출, 광고②③·쿠팡·관련글 12개는 하단 그대로. 광고도 댓글도 안 희생하면서 다음 글만 끌어올립니다.

3. 실험 2 · "첫날 1차 목표" — 가입·설치 동선 ⭐ 2순위

🧩 왜 가입·설치를 한 실험으로 AS-IS 검증 결과 비로그인 = 가입 3겹 / 로그인 = 설치 CTA로 갈립니다. 둘은 같은 자리(PostCTA)에서 로그인 여부로 바뀌므로, "신규에게 무엇을 먼저 들이밀까"라는 하나의 동선입니다.
📖 "압박 ↓"의 정확한 뜻 (창업자 질문 반영) v2의 "첫 3화면 압박↓"이 모호했습니다. 정확히는 — 비로그인이 3화면 이상 둘러보기 전까지, ③풀스크린 가입 띠배너(SignupPromptBanner)의 발동을 미룬다는 뜻입니다. ①정체성 배너·②"1초 가입" 인라인은 화면을 가리지 않으니 그대로 유지. 가장 침투적인 ③만 뒤로 미뤄 둘러보기를 먼저 시키는 것.
