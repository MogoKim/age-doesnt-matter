1-1. 4축 분류 (Desire Type)

코드이름정의예시 신호

demerit 디메리트 현재의 마찰/불만/장벽 "광고가 너무 많다", "설명이 어렵다"

1-2. 욕망 카테고리 (Desire Category) — 13개

HEALTH
🏥 건강
갱년기, 관절, 당뇨, 혈압, 약, 병원, 수술, 영양제
"더 나빠지면 어떡하나", "나만 이런 건가"
RELATION
🤝 관계/외로움
친구 없다, 서럽다, 혼자다, 대화 상대, 공감
"친구가 없다", "말 걸 곳이", "이해해주는 사람"
MONEY
💰 돈/재정
생활비 절약, 남편 은퇴, 현금흐름, 연금, 보험
"어떻게 아껴요", "생활비 얼마면", "통신비 줄이기"
RETIRE
🌅 은퇴/인생2막
퇴직, 할 일, 의미, 우울, 새 시작, 은퇴 계획
"퇴직하고", "뭘 해야 할지", "허전하다"
FREEDOM
🕊️ 자유/독립
가족·의무에서 벗어나 나만의 공간과 시간
"혼자 있고 싶다", "나만의 시간", "나를 위한"
DIGNITY
👑 존중/자존감
무시당하는 느낌, 고부갈등, 나이 차별
"무시당한다", "아직 할 수 있는데", "나이 차별"
FAMILY
👨‍👩‍👧 가족
자식 걱정, 남편 갈등, 손주 자랑, 시댁
"자식이", "남편이", "손주", "며느리가"
JOB
💼 일자리/부업
알바, 파트타임, 자격증, 재취업, 부업
"알바", "자격증", "재취업", "뭔가 해야 할 것 같다"
HOBBY
🎨 취미/여가
등산, 여행, 배움, 시간 활용, 귀농귀촌 (직접 참여)
"등산", "텃밭", "뭔가 배워보고 싶다"
MEANING
✨ 삶의 의미
이 나이에, 살아있다, 감사하다, 보람
"이 나이에", "살아있다는 게", "감사하다"
LEGACY
📜 남기기/기억
내가 죽으면, 자식에게 해주고 싶다
"내가 죽으면", "자식들한테 해주고 싶다"
CARE
🏥 돌봄/간병
부모 간병, 배우자 돌봄, 내가 아프면
"부모님 간병", "내가 아프면 누가", "요양원"
ENTERTAIN
🎵 연예/팬덤
드라마, 트로트, 연예인 팬덤 (소비형)
"임영웅", "트로트", "드라마 결말", "콘서트"
구분 원칙
HOBBY vs ENTERTAIN: HOBBY는 내가 직접 몸을 움직여 하는 활동. ENTERTAIN은 콘텐츠·연예인을 팬으로서 보고 즐기는 것.
CARE vs FAMILY: FAMILY는 감정적 관계(사랑, 갈등). CARE는 돌봄 부담과 의존 공포.
FREEDOM: 50-60대 여성 특유. 기존 카테고리에 잘 포착되지 않던 욕망.
ENTERTAIN > 10%: 전체 수집 글의 10% 이상이면 엔터 전담 페르소나(EN1~EN5) 활성화.

1-4. 긴급도 (Urgency Level) — 1~5

점수수준신호예시
5 즉각적 고통/위기 지금 당장 "오늘 병원에 가야 하는데", "당장 돈이 없다"
4 근 미래 불안 이번 주~이번 달 "다음 달이 걱정", "이 증상이 점점 심해진다"
3 중기 고민 올해 안에 "올해 안에 해야 할 것 같다"
2 막연한 걱정 언젠가 "나중에 어떡하지", "막막하긴 한데"
1 일상적 관심 정보 공유/수다 정보 공유, 취미 이야기, 일상 수다

긴급도는 보수적으로 설정 — 확실하지 않으면 낮게 (1-2).

1-5. 커뮤니티 신호 (communitySignal)

코드의미신호
question무언가 물어보러 왔다 (정보 갈증)"어떻게 해요?", "추천해 주세요"
complaint하소연/불만 배출"너무 힘들어요", "들어줘요"
confession고백/털어놓기"사실은...", "아무도 몰랐는데"
recommendation경험 공유/추천"이거 해봤는데", "꼭 해보세요"
celebration자랑/기쁨 나누기"드디어!", "해냈어요!"

1-6. 분석 원칙

  1. 글 + 댓글 같이 본다: 제목 < 본문 < 댓글 순으로 진짜 감정이 강해짐
  2. 정보글보다 감정글 우선: 건강 정보 공유보다 "이 증상 나만 이렇나요?"가 욕망을 더 잘 보여줌
  3. desireType은 하나만: 가장 지배적인 것 하나 (big_desire / need / want / demerit)
  4. emotionTags는 복수 가능: 최대 3개
  5. desireCategory도 하나: 가장 지배적인 카테고리 하나
  6. urgencyLevel은 보수적으로: 확실하지 않으면 낮게 (1-2)
  7. communitySignal + ageSignal: 없으면 unknown/null 허용

2-4. 5 Whys — 욕망의 뿌리

Q. 왜 인생설계 글에 조회수 9,253이 나오는가?
  1. 실버타운·주거 고민이 많다
  2. 노후를 어디서 어떻게 살지 모른다
  3. 퇴직 후 삶의 지도가 없다
  4. 인생 다음 챕터에 대한 사회적 준비·문화·대화가 없다
💡 핵심: "이 나이에 어떻게 살아야 하는지 아무도 안 알려줬다"
Q. 왜 건강 글은 많이 쓰지만 반응이 낮은가?
  1. 건강 정보는 공감보다 검색 목적
  2. 갱년기·증상은 '나만의 비밀'처럼 느껴짐
  3. 체면 장벽 — 아프다고 드러내기 어려움
  4. 건강 불안의 본질은 "내가 혼자 이걸 감당해야 하나?"
💡 핵심: 건강 뒤에는 고립감(RELATION)이 있다
Q. 왜 외로움 키워드는 글 수가 적은데 반복 등장하는가?
  1. 외로움을 직접 표현하면 체면 손상
  2. 외로움은 건강/유머/일상 글 속에 녹아있다
  3. "혼자다"라고 말하는 게 아니라 글을 쓰러 온다
  4. 커뮤니티 자체가 외로움 해소 행동
💡 핵심: 우나어에 오는 행위 자체가 RELATION 욕망의 표현

2-5. 확정된 욕망 순위

2026-04-15 창업자 확정 — 고객 상태 분석 기반 순위 재정렬

#1

RELATION — 외로움·느슨한 연결

체면 뒤에 숨은 가장 깊은 욕망. 커뮤니티에 오는 행위 자체가 이 욕망의 표현.

#2

HEALTH — 건강 염려·악화 공포·현상 유지 ↑ #4→#2

갱년기·체력 저하를 지금 직접 몸으로 느끼고 있음. 더 나빠지지 않았으면 + 위로받고 싶다.
3층 구조: 정보 욕구(표면) → 공감 욕구(중간) → 위로 욕구(핵심, RELATION과 연결)

#3

MONEY — 절약 지혜·현금흐름 불안·남편 은퇴 재정 재편

자산은 있지만 현금흐름이 조여듦. 절약 지혜 + 생활비 현실이 실전 이슈.
보험 정리·통신비·자녀 지원 기준·남편 용돈 재협상.

#4

RETIRE — 인생의 의미·다음 챕터 ↓ #2→#4

퇴사·주거·인생2막 (조회수 여전히 높음). 이제야 나 자신을 보기 시작하는 시기.

+

FREEDOM — 나만의 시간·독립 (강도 UP)

남편 은퇴 + 자녀 독립 = 이 시기 특유의 폭발. 처음으로 자신을 위한 무언가를 원함.
"남편이 맨날 집에 있어서 숨막혀요"

+

DIGNITY — 자존감·인정 (신규 강조)

전직 정규직→전업주부 공백. 능력 있었는데 지금은 아무것도 아닌 것 같은 느낌.
남편 은퇴 후 발언권 약화 느낌.

3-1. 서비스 정체성

서비스명우리 나이가 어때서: 일과 삶을 잇다
줄임말우나어
한 줄 설명제2의 인생을 꿈꾸는 5060을 위한 일자리 + 커뮤니티 플랫폼
본질일(Job)로 들어와서, 사람(Community)으로 남는 구조
작은 소일거리부터 하루의 이야기까지,
당신의 새로운 '내 일'과 든든한 '내 편'을 잇습니다.

MISSION: 50·60대가 다시 사회와 연결되고, 자신만의 속도로 새로운 삶을 시작할 수 있도록 돕는다.

VISION: 나이가 가능성의 한계가 되지 않는 세상, 누구나 설레는 마음으로 두 번째 전성기를 맞이하는 라이프 플랫폼

4-2. 브랜드 톤 & 보이스

페르소나: 옆자리 친구

"같은 나이, 같은 고민, 먼저 공감해주는 따뜻한 동네 친구"

소통 순서 (절대 준수)
① 공감 → ② 위로 → ③ 응원 → ④ 정보
✅ 말해야 하는 것
  • 존댓말 필수 (~해요, ~입니다)
  • 어렵지 않은 단어, 전문용어 풀어쓰기
  • 결론(이득)은 무조건 두괄식
  • 밝고 따뜻하게 — 응원과 긍정은 적극적으로
  • 위로하고 '할 수 있다' 믿어준다
❌ 절대 금지 표현
  • '시니어', '어르신', '노인' → 50·60대, 우리 또래
  • 조언하거나 가르치는 톤
  • 정치·세대갈등 언급
  • '요즘 젊은이들은', '우리 때는'
  • 공감 없이 정보부터 던지는 것
  • TMI, 갓생, 핵인싸 등 젊은 세대 표현

  4. 에이전트 해석 불가 → 창업자에게 질의

5-2. 3가지 유지 메커니즘

🔒
① Git 버전 관리

constitution.yaml · NORTH_STAR.md 모든 변경을 git commit으로 기록.
에이전트는 이 파일에 write 불가 (읽기 전용).
변경 절차: 창업자가 직접 편집 → commit → 다음 에이전트 실행 시 자동 반영

💉
② System Prompt 주입

에이전트가 실행될 때마다 constitution.yaml을 새로 읽어 System Prompt 첫 줄에 주입.
자동 반영: constitution.yaml이 바뀌면 다음 실행부터 즉시 적용. 에이전트 재시작 불필요.

👁
③ CEO 에이전트 감시

매일 모닝 사이클: 전날 모든 에이전트 액션 로그 읽기 → 헌법 기준 위반 여부 체크
위반 시: 해당 에이전트 일시 정지 + 텔레그램 창업자 즉시 알림 + 승인 전까지 비활성화
