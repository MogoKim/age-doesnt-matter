## 문서 구성 원칙

각 기획서는 아래 섹션을 포함한다:

1. **목표** — 이 기능이 달성하려는 비즈니스 목표
2. **배경** — 왜 만들었는가, 어떤 문제를 해결하는가
3. **세부 기획** — 파이프라인·스케줄·필터 규칙 등 운영 세부사항
4. **관련 링크** — 코드 파일, GA 워크플로우, 외부 URL
5. **수정 히스토리** — 기능 변경 이력 (날짜·변경 내용·이유)
6. **이슈 히스토리** — 발생한 문제, 원인, 해결 방법

---

## 이슈 발생 시 체크리스트

1. 해당 기능의 기획서 **이슈 히스토리** 섹션 확인 — 과거 유사 이슈 있는지
2. BotLog DB 조회 — 최근 실행 기록 확인
3. GA 워크플로우 실행 이력 확인 (`gh run list --workflow=...`)
4. 원인 파악 → 수정 → 이슈 히스토리에 기록
