목적과 배경

우나어는 50·60대의 외로움(RELATION)·건강 염려(HEALTH)·재정 불안(MONEY)·인생 2막(RETIRE)을 중심으로 설계된 AI 운영 커뮤니티입니다. 현재 MAU 21명에서 Q2 목표 500명을 향해 가는 초기 단계에서, 시드봇 79명의 페르소나가 "실제 사람처럼" 글·댓글을 작성해 초기 활성화를 이끌고 있습니다.

핵심 Why — 크롤링이 필요한 이유

네이버 카페 크롤링은 단순 콘텐츠 수집이 아닌 욕망 인텔리전스 시스템의 핵심 입력값입니다. 실제 50·60대가 쓴 글에서 감정·욕망·공포·페르소나를 추출하여, 시드봇의 오늘 활동 배율(0.5×~2.0×)과 큐레이션 주제를 결정합니다.

데이터 흐름 요약
네이버 카페 2곳
  → 크롤링 (하루 4회)
  → CafePost DB (글·댓글·욕망태깅)
  → psych-analyzer (욕망지도)
  → DailyBrief (오늘의 핫토픽)
  → 시드봇 79명 (활동 배율 결정)
  → 큐레이션 9건/일 발행

시스템 아키텍처

크롤링 파이프라인은 Mac 로컬 환경(launchd plist)과 GitHub Actions(GHA)가 이중으로 연동됩니다. Mac이 꺼져 있을 경우 GHA가 스마트 fallback으로 큐레이션을 보장합니다.

실행 레이어 구분

Mac plist (LOCAL) Playwright 필요 작업
→ 크롤링, 세션 관리, psych 분석, 큐레이션

GitHub Actions Playwright 불필요 작업
→ DailyBrief fallback, 큐레이션 fallback, 댓글 파동

락 파일 방식 (/tmp/unao-crawler.lock)
TTL: 30분 초과 시 자동 해제

GHA 스마트 fallback
BotLog에서 당일 CONTENT_CURATE 기록 조회 → ≥1건이면 GHA 큐레이션 skip

6단계 파이프라인 상세

크롤에서 DailyBrief까지 6단계가 순차 실행됩니다. DEEP(11:30 FULL)은 전 단계 실행, QUICK은 심리분석 스킵, CRAWL-ONLY(08:30)는 1~4단계만 실행합니다.

1
카페 목록 크롤링 (전체글보기) DEEP CRAWL-ONLY QUICK
crawler.ts — collectAllArticleUrls()
전체글보기 URL에서 페이지 순회하며 게시글 URL 수집. MAX(articleId) 이후 글만 크롤 (증분 방식). minSave=20건 미만이면 다음 페이지 계속. 최대 수집 목표: 50건/카페 (FULL/CRAWL-ONLY), 15건/카페 (QUICK).

wgang URL: https://cafe.naver.com/f-e/cafes/29349320/menus/0?viewType=L&page=N
dlxogns01 URL: https://cafe.naver.com/f-e/cafes/23676262/menus/0?viewType=L&page=N
2
게시글 본문 수집 DEEP CRAWL-ONLY QUICK
crawler.ts — collectPostContent()
각 게시글 URL 방문 → 제목·본문·이미지·댓글·작성일 추출. 상위 댓글(topComments) likeCount 기준 정렬 수집. 이미지: R2 업로드 (scraped/{postKey}/{index}.ext). 중복 체크: @@unique([cafeId, articleId]). isUsable 필터: qualityScore ≥ 30 (minUsable).
3
심리분석 — Claude Haiku DEEP/FULL 전용
psych-analyzer.ts — analyzePsychology()
욕망 카테고리 20개 분류 (기존 13 + BEAUTY/DIGITAL/FOOD/SPIRITUAL/HOUSING/FASHION/PET 추가). 출력: 감정 태깅, 욕망 분류, 공포 분류, 페르소나 매핑. 배치: 수집 게시글 전량 처리 (~50건). 비용: ~$0.0008/1K input, $0.0032/1K output.
4
DB 저장 DEEP CRAWL-ONLY QUICK
crawler.ts — savePosts()
CafePost INSERT (cafeId, articleId, postUrl, title, content, images, topComments, psychTags, killerScore, viralType). usedAt: NULL (큐레이션 참조 시 갱신). articleId: 네이버 순차 번호 (dedup 기준). killerScore: 참여도 55% + 콘텐츠 길이 20% + 게시판 우선순위 15% + 미디어 5% + 최신성 5%.
5
트렌드 분석 — Claude Sonnet-4-6 FULL 전용
trend-analyzer.ts — analyzeTrends()
카테고리별 CafeTrend 집계 + 주간 트렌드 변화 감지. postSummaries에 댓글 분위기·베스트댓글 포함. 출력: hotTopics (시드봇 활동 배율 0.5×~2.0× 결정 기준). DailyBrief의 욕망 지도 데이터 제공.
6
DailyBrief 생성 — Claude Sonnet-4-6 FULL 전용 (11:30)
daily-brief.ts — generateDailyBrief() / loadTodayBrief()
당일 수집 전체 요약 → DailyBrief 테이블 저장 + Slack #대시보드 전송. loadTodayBrief(): 오늘→어제→최근 fallback으로 brief 조회. 15:30/21:30 CRAWL-CURATE 모드는 생성 없이 읽기만 (재생성 없음). brief 없을 경우 CafeTrend DB 직접 조회로 hotTopics 구성.

CRAWL-CURATE 추가 단계 (15:30 · 21:30)

6단계 완료 후 content-curator.ts 실행 → 큐레이션 3건 발행. loadTodayBrief()로 11:30 생성 brief 재활용 (재생성 없음, AI 비용 0). 발행 후 /api/internal/comment-wave 호출 → wave1 즉시 댓글 1분 이내 생성. CommentWaveQueue에 wave2~4 스케줄 저장 → GHA */5분 cron이 처리.

큐레이션 선택 기준

② 날짜 필터: 발행일 기준 48시간 이내 원문만
③ usedAt IS NULL (중복 참조 방지)

댓글 파동 시스템 (Comment Wave)

큐레이션 게시글 발행 직후 시드봇이 단계적으로 댓글을 달아 초기 인게이지먼트를 만드는 시스템입니다. 알고리즘 활성 기준(첫 댓글 10분 이내)을 충족하기 위해 설계되었습니다.

단계타이밍실행 주체처리 방식
wave1 발행 후 1분 plist (content-curator.ts 호출) /api/internal/comment-wave 직접 호출
wave2 발행 후 5분 GHA wave-processor (*/5분) CommentWaveQueue wave2Done=false 처리
wave3 발행 후 30분 GHA wave-processor (*/5분) CommentWaveQueue wave3Done=false 처리
wave4 발행 후 60분 GHA wave-processor (*/5분) CommentWaveQueue wave4Done=false 처리

댓글 생성 원칙

원문 보존

topComments 원문 90% 보존. 어미·말투만 페르소나 스타일로 변환. 고유명사·수치·지명 100% 유지.

페르소나 선택

COMMENTER_PERSONA_IDS: 50명 (A~AY, D 제외). 글쓴이 페르소나는 wave에서 자동 제외. 글별 랜덤 3명 선택.

CommentWaveQueue TTL

expiresAt = createdAt + 60시간
만료 초과 항목: GHA wave-processor 실행 시 자동 정리 (DELETE expiresAt < NOW()). 게시글 삭제 후에도 고아 데이터 누적 방지.

쿠키·세션 관리

네이버 카페 접근에 필요한 쿠키는 두 종류입니다. session-manager.ts가 만료 감지·자동갱신·경보를 담당합니다.

NID_SES (단기 세션)
TTL: 짧음 (수일~수주)
처리: 잔여 ≤5일 시 자동갱신
알림: 갱신 성공 → Slack #시스템 info
NID_AUT (장기 인증)
TTL: 30~60일
처리: 잔여 ≤30일 → 매일 2채널 경보
처리: 잔여 ≤60일 → 시스템 경보 시작
만료 시: 즉시 종료 + 긴급 Slack 알림 + 수동 재인증 필요

쿠키 만료 시 대응 절차

🍪 네이버 쿠키 수동 갱신 절차
1
Chrome 브라우저에서 네이버 로그인 (wgang 접근 가능한 계정)
2
개발자 도구(F12) → Application → Cookies → naver.com에서 NID_AUT, NID_SES 값 복사
3
.env.local의 NAVER_COOKIES 업데이트 (JSON 배열 형식)
4
GitHub Secrets NAVER_COOKIES도 동일값으로 업데이트
5
수동 크롤 테스트: PIPELINE_MODE=crawl-only npx tsx agents/cafe/run-pipeline.ts
6
Slack #시스템에서 크롤 성공 메시지 확인
쿠키 만료 예방 session-manager.ts가 NID_SES를 자동갱신하지만, NID_AUT는 수동 갱신 필수입니다. Slack #시스템 경보를 받으면 즉시 위 절차를 실행하세요. 30일 이상 방치 시 크롤 전면 중단됩니다.

🔴 쿠키 만료 / 로그인 감지 (Slack #시스템 URGENT 알림)
1
크롤링 즉시 중단됨. Slack #시스템 긴급 알림 수신 확인

4
PIPELINE_MODE=crawl-only npx tsx agents/cafe/run-pipeline.ts 실행하여 테스트
5
Slack #시스템에서 CAFE_CRAWL SUCCESS 메시지 확인
🟡 큐레이션 0건 (당일 발행 없음)
1
우나어 게시판에서 오늘 큐레이션 글 없음 확인
2
BotLog 확인: SELECT * FROM "BotLog" WHERE "botType"='CAFE_CRAWLER' AND DATE("executedAt")=TODAY ORDER BY "executedAt" DESC LIMIT 10
3
isUsable 통과 게시글 확인: SELECT COUNT(*) FROM "CafePost" WHERE "usedAt" IS NULL AND "qualityScore" >= 30 AND "createdAt" > NOW() - INTERVAL '48 hours'
4
0건이면 크롤 먼저 실행 후 재시도: PIPELINE_MODE=crawl-curate npx tsx agents/cafe/run-pipeline.ts
🔵 plist 스케줄 미실행 (Mac 재부팅 후)
1
plist 상태 확인: launchctl list | grep unao
2
전체 재로드: launchctl unload ~/Library/LaunchAgents/com.unao.cafe-crawler-*.plist && launchctl load ~/Library/LaunchAgents/com.unao.cafe-crawler-*.plist
3
재로드 후 상태 재확인: launchctl list | grep unao (PID가 0이 아니면 실행 중)
🟠 락 파일 충돌 (이전 실행이 비정상 종료)
1
락 파일 확인: ls -la /tmp/unao-crawler.lock
2
30분 TTL 초과 시 자동 해제됨. 즉시 해제 필요 시: rm /tmp/unao-crawler.lock

실행 중인 프로세스 확인: ps aux | grep crawler

09:30 GHA Brief Monitor가 자동으로 Slack 경고 발송 여부 확인

유용한 진단 쿼리

-- 오늘 크롤 현황
SELECT action, status, "itemCount", "executedAt"
FROM "BotLog"
WHERE "botType" = 'CAFE_CRAWLER'
  AND DATE("executedAt") = CURRENT_DATE
ORDER BY "executedAt" DESC;

-- isUsable 통과 게시글 수 (큐레이션 가능 원본)
SELECT COUNT(*) FROM "CafePost"
WHERE "usedAt" IS NULL
  AND "qualityScore" >= 30
  AND "createdAt" > NOW() - INTERVAL '48 hours';

-- 카페별 최신 articleId (증분 크롤 기준점)
SELECT "cafeId", MAX("articleId") as max_id, COUNT(*) as total
FROM "CafePost"
GROUP BY "cafeId";

-- CommentWaveQueue 대기 현황
SELECT postId, wave1Done, wave2Done, wave3Done, wave4Done, expiresAt
FROM "CommentWaveQueue"
WHERE "expiresAt" > NOW()
ORDER BY "createdAt" DESC LIMIT 20;

GitHub Actions - Public repo 무제한 $0 $0
합계 ~$168/월
