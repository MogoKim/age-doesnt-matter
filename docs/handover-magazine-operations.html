keyword queue 설명

keyword-universe.json — 구글 자동완성 등으로 모은 약 1,300개 키워드 우주(발행 후보 약 1,188개). 각 키워드에 클러스터·검색의도·민감도·발행정책(publishPolicy)·점수가 붙어 있습니다. gitignore 대상.

selector (keyword-queue.ts) — 후보를 우선순위로 정렬해 다음 키워드를 고릅니다.

규칙의미
클러스터 라운드로빈건강·관계·일자리 등 주제군을 번갈아 → 하루 2편이 서로 다른 주제군
클러스터 내 점수순같은 주제군 안에서는 수요·적합도 높은 것 우선
서브토픽 분산같은 세부주제가 7일 안에 반복되지 않게 분산
publishPolicy 필터publish / publish_softened_title만 후보. no_publish_*는 절대 제외

이중 가드 · state 소비 추적

  • publishPolicy 이중 가드: 선택 단계(publish/softened만) + 발행 직전(no_publish면 failed_no_publish_guard 차단)
  • state agents/magazine/data/keyword-queue-state.json(DB 아님, gitignore): consumed = published + skipped_duplicate / failed_*(생성·이미지·짧음·가드) = 미소비 → 재시도 / 이벤트 최근 500개
  • ⚠️ state 손상/백업 정책은 미비 — 후속 과제(§11)

이미지 운영 + 품질 가드

이미지는 ChatGPT입니다(plist IMAGE_GENERATOR=chatgpt, morning/late 동일). ChatGPT 웹 UI를 자동화해 생성 후 R2 업로드.

저해상/중복 재발 방지 (커밋 3cf88f0, launchd 경로 적용) 이전에 본문 이미지가 히어로의 저해상 미리보기(≈512px WebP)로 박히던 결함을 차단했습니다. ① scraper가 naturalWidth/Height 800px+ full-res만 채택(미리보기 무시), ② generator가 히어로↔본문 동일·저해상 판정 시 본문 이미지 1회 재생성 → 그래도 불량이면 본문 이미지 없이 발행(degrade). 발행은 롤백하지 않습니다.
이미지 타입처리
PERSON_REAL (인물)ChatGPT 전용. 실패 시 fallback 없이 발행 보류(외국 스톡 방지)
FOOD / SCENE / OBJECT실패 시 Unsplash 대체 가능
ILLUSTRATION현재 비활성(레거시 DALL-E 경로)
본문 이미지 장수현재 2장(히어로 + 본문 1). 3장 확장은 후속(비용·가드 다중적용 필요)
ChatGPT 세션 만료 시이미지가 실패해 인물 글은 보류되고 Slack 알림이 옵니다. chatgpt.com 재로그인 필요(프로필 ~/.chrome-chatgpt-profile).

참고(과거): 과거에는 Gemini로 이미지를 만들었으나 현재 값은 ChatGPT입니다(레거시). morning plist 주석에 옛 gemini 표기가 남아 있을 수 있으나 실제 환경변수 값은 chatgpt라 동작엔 무관합니다.

editorial v2 (flag · 구조 · 시범)

editorial v2 = 가독성 개선 템플릿. flag 뒤에 있고 기본 OFF. 코드는 main(2ea8278)·poc(6fdac2c) 양쪽에 배포됐으나, MAGAZINE_EDITORIAL_V2_ENABLED가 꺼져 있으면 기존 v1로 발행됩니다. 발행은 launchd(poc 체크아웃) 경로이므로 v2 동작은 poc 코드 + flag로 결정됩니다.

v2가 추가하는 것 (v1 대비)

  • 직접답변 박스 — 제목의 검색 질문에 첫 화면에서 곧바로 답
  • 핵심 요약 3줄 — 글 요점을 스캔 가능하게
  • 기존 본문 sections + FAQ(details/summary) 유지
  • 40대 50대 60대 여성 커뮤니티 연결박스 (외부 검색 유입자용)
  • 제외: <table>(sanitize 미허용) · 이미지 3장 · 본문 중간 광고

안전장치

  • flag OFF면 v1과 byte-identical(v1 함수 무변경, 순수 추가)
  • 직접답변/요약 파싱 실패 → 해당 박스만 생략(degrade), 발행 계속
  • v2 빌드 throw → v1 fallback, 발행 롤백 안 함
  • inline-style만 사용 → sanitizeMagazineHtml 화이트리스트 통과(table/position/list-style 등 미사용)
시범 상태 (2026-06-23 기준) .env.local에 MAGAZINE_EDITORIAL_V2_ENABLED=true가 설정됐습니다(로컬 시범, gitignore — Vercel/GHA env 미설정). 다음 launchd 발행 1편이 v2로 생성될 예정이며, 아직 v2 발행글 실측은 안 됨. QA 후 유지/롤백 판단(§12).

예외 처리 Q&A

Q. 주제(키워드)가 다 떨어지면?
consumed에 없는 키워드를 순서대로 씁니다. 후보 약 1,188개 = 하루 2편 기준 1.6년치. 그래도 비면 fallback(시리즈/트렌드/욕망지도)으로 넘어가 빈손으로 끝나지 않습니다.
Q. Mac이 꺼져 있으면?
그 시각엔 발행이 안 됩니다. 키워드는 소비되지 않고 다음 실행에서 이어 씁니다. 빠진 분량을 다음 날 몰아서 보충 발행하지는 않습니다(세션당 1편 유지).
Q. 이미지가 저해상/중복으로 나오면?
scraper가 800px+ full-res만 채택하고, generator가 히어로↔본문 동일·저해상 판정 시 본문 이미지를 1회 재생성, 그래도 불량이면 본문 이미지 없이 발행(degrade). 인물 이미지 실패는 발행 보류.
Q. v2가 켜졌는데 v2 블록이 안 보이면?
직접답변/요약 파싱 실패 시 해당 박스만 생략됩니다(정상 degrade). v2 빌드 자체가 실패하면 v1으로 fallback해 발행은 계속됩니다.
Q. 글 생성/본문 짧음/제목 중복은?
생성 실패=failed_generation(미소비·재시도) / 1,500자 미만=failed_body_short(미소비) / 제목 중복=skipped_duplicate(소비, 재시도 안 함). 기존 중복가드 유지.
Q. no_publish 키워드가 섞이면?
선택·발행 직전 이중 가드로 차단됩니다. 발행될 수 없습니다.
Q. v2/queue를 끄려면?
.env.local에서 MAGAZINE_EDITORIAL_V2_ENABLED 또는 MAGAZINE_KEYWORD_QUEUE_ENABLED를 =false로 바꾸면 즉시 이전 동작으로 복귀(기존 글 무영향).

배포 경로 (main↔poc) · 위험/주의
