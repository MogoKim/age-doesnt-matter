시스템 개요

매거진은 AI 완전 자동 발행 시스템입니다. 사람 개입 없이 하루 2회 기사를 생성·발행합니다.

launchd (macOS 스케줄러)
└─ 11:00 KST (morning 세션) → 기사 생성 + DB 발행 + JSON 저장
└─ 14:00 KST (late 세션)    → 기사 생성 + Slack 통합 알림 + JSON 삭제
항목값
DB 테이블Post (boardType = 'MAGAZINE')
봇 작성자editorUserId — 매거진 전용 봇 계정 (정순씨)
Slack 채널#magazine (C0ARZET1X63) — 일일 발행 알림
일일 최대 발행3편 (hardcoded)
이미지 엔진Gemini Imagen (Playwright 스크래퍼) → Unsplash 폴백
기사 생성 모델Claude Sonnet (월~토) / Claude Opus (일)
실행 환경macOS 전용 (launchd + Playwright) — GitHub Actions 비활성

실행 파이프라인 (코드 흐름)

3-1. 전체 흐름

launchd 트리거 (11:00 or 14:00 KST)
→ launchd-alert.sh (래퍼: 실패 시 #알림-시스템 Slack)
→ tsx local-magazine-runner.ts
1. dotenv 로드 (.env.local) ← 반드시 최우선
2. R2 dynamic import (dotenv 이후 — S3Client 초기화 보장)
3. SESSION_TIME 정규화: late → evening, 나머지 → morning
4. magazine-generator.ts 호출 → articles[] 반환
5. daily JSON 관리
morning: morningArticles 저장
evening: eveningArticles 추가 → Slack 전송 → 파일 삭제

3-2. 기사 선정 우선순위 (magazine-generator.ts)

순위소스score조건
1순위시리즈 연재 편10월요일만 (dayOfWeek === 1)
2순위GEO 시드 주제7~10지오시드 에이전트 결과
3순위트렌드 분석 주제7~9카페 트렌드 에이전트 결과
4순위욕망 지도 폴백71~3 모두 없을 때 (HEALTH/MONEY/RELATIONSHIP 등)

필터: score ≥ 7점  +  isSimilarTitle() 통과 (최근 30일 제목 첫 3단어 중 2개 미만 겹침)  +  일일 3편 한도 미초과

3-3. 기사 생성 모델

요일모델이유
월~토claude-sonnet-4-6 (HEAVY)일반 기사 — 비용 효율
일요일claude-opus-4-6 (STRATEGIC)주간 심층 기획 기사

3-4. 이미지 생성 흐름

IMAGE_GENERATOR=gemini (plist 환경변수)
→ local-image-generator.ts
→ gemini-scraper (Playwright, Chrome ~/.chrome-gemini-profile)
성공: buffer → R2 업로드 → pub-*.r2.dev URL 반환
실패/타임아웃(120초): null 반환
→ Unsplash 폴백 (PERSON_REAL / FOOD_PHOTO / SCENE_PHOTO / OBJECT_PHOTO만)
→ fetchUnsplashPhoto() → R2 업로드 (2회 재시도) → URL
실패: null
→ null 최종 반환 → 히어로 이미지 없음 → 기사 발행 보류
이미지 타입처리
PERSON_REAL한국 여성 4050 실사 Gemini 프롬프트
FOOD_PHOTO / SCENE_PHOTO / OBJECT_PHOTO카테고리별 Gemini 프롬프트 → Unsplash 폴백
ILLUSTRATION비활성 DALL-E 연동 주석 처리 — 히어로 없으면 발행 보류
