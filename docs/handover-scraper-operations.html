01 — 배경

왜 만들었는가

우나어는 50~60대 여성이 공감하는 콘텐츠로 채워져야 한다.
그런데 초기에는 사용자가 없으니 게시글이 없고, 게시글이 없으니 사용자가 안 온다.
이 악순환을 끊기 위해 외부 커뮤니티에서 좋은 글을 가져와 자동으로 올리는 시스템을 만들었다.

03 — 창업자 역할

창업자가 하는 일 — 이게 전부다

A열에 URL만 넣으면 끝. 나머지는 전부 자동이다.
창업자가 "이 글 좋다"고 판단한 URL을 Google Sheets에 붙여넣으면, AI가 판단하고, 봇이 올리고, 파동이 HOT을 만든다.

Google Sheets 탭 구조

탭 이름어디에 올라가는가파동 강도
사는이야기우나어 사는이야기 게시판약한 파동 (일반)
웃음방우나어 웃음방 게시판약한 파동 (일반)
사는이야기_화제성사는이야기 + HOT 탭 자동 승격강한 파동 — 1분 이내 시작
웃음방_화제성웃음방 + HOT 탭 자동 승격강한 파동 — 1분 이내 시작

입력 방법

A열에 URL만 넣으면 된다. B열 상태(Status)는 자동으로 PENDING이 된다.

예시: 창업자가 오전 10시에 URL 5개를 넣으면 → 12:00 KST에 자동 처리된다.

자동 처리 단계 (전체 흐름)

1
PENDING 상태 URL 감지
시트에서 B열 = PENDING인 행만 가져온다. 처리 시작 시 즉시 PROCESSING으로 변경 (중복 처리 방지)
2
AI 관련성 판단 (Claude Haiku)
50~60대 여성 관련성 0~10점 측정 → 6점 미만이면 SKIPPED_LOW_SCORE로 처리하고 G열에 사유 기록. 다음 URL로 넘어감
3
AI 카테고리 분류 + 제목 최적화
카테고리 자동 분류 (유머/일상/건강/요리/여행/기타). 제목이 너무 짧거나 ㅋㅋ 등 특수문자만이면 더 나은 제목으로 자동 개선
4
Playwright 브라우저로 원본 접속 + 댓글 수집
해당 URL에 자동으로 접속해 본문 글 추출, 이미지 추출, 원본 댓글 최대 10개 수집 (AJAX 로딩 최대 5초 대기). 수집된 댓글은 파동 댓글 AI 각색에 활용
5
이미지 R2 클라우드에 자동 업로드
외부 도메인 이미지를 우나어 서버(Cloudflare R2)에 재업로드. 원본 도메인 차단 시에도 이미지 정상 표시됨
6
우나어 사이트에 게시글 자동 생성
5명의 AI 페르소나 중 카테고리에 맞는 1명이 게시자로 표시. 출처 URL 포함. boardType은 시트 탭에서 결정
7
화제성 탭이면 — 즉시 파동 시작
+1분: 좋아요 10명 자동 클릭  |  +3분: 공감 댓글 5개 자동 생성
+6분: 반박/다른 의견 댓글 3개  |  +10분: 반전·마무리 댓글 5개
→ HOT 탭 자동 승격
8
시트 상태 업데이트 + Slack 알림
성공 → F열에 게시글 URL + PUBLISHED. 실패 → G열에 에러 + FAILED. Slack #로그 채널에 결과 전송
05 — AI 역할

AI가 하는 4가지 판단

2026-05-15부터 Claude Haiku AI가 스크래퍼에 추가됐다. 이전에는 단순 키워드 매칭이었지만, 이제는 AI가 문맥을 이해해서 판단한다.

판단 1 — 관련성 점수 (0~10점)

창업자가 URL을 넣어도 AI가 먼저 "이 글이 우리 타겟에 맞는가"를 본다.

우리와 무관 → SKIP

AI도 실수할 수 있다. 좋은 글인데 SKIPPED됐다면 → D열에 카테고리를 직접 지정하고 B열을 다시 PENDING으로 바꾸면 재처리된다.

판단 2 — 카테고리 분류

기존 키워드 방식의 문제
"남편이 웃음을 잃었어요" → 웃음방(HUMOR)으로 잘못 분류
"건강한 유머 생활" → 건강? 유머? 딕셔너리 순서대로 결정

AI 분류 (문맥 이해)
"남편이 웃음을 잃었어요" → 사는이야기(고민)으로 정확 분류
창업자가 D열에 직접 지정했으면 그걸 우선 따름


판단 3 — 제목 최적화 (조건부)

조건처리예시
제목 10자 미만 또는 특수문자만 AI가 더 나은 제목 자동 생성 (최대 25자) "ㅋㅋㅋ" → "남편 자신감이 너무 넘쳐서 웃음 터진 일"
제목이 충분한 경우 원본 제목 그대로 유지 —
창업자가 C열에 직접 입력 창업자 제목 최우선 적용 —

판단 4 — 원본 댓글 AI 각색 2026-05-16 신규

스크랩 시 원본 사이트 댓글을 최대 10개 자동 수집한다.
수집된 댓글은 파동 댓글 생성 시 AI가 50~60대 말투로 각색해 활용한다.
이미지만 있고 본문 텍스트가 없는 글도 원본 댓글을 보고 맥락을 이해해 자연스러운 댓글을 만든다.

게시글 유형파동 댓글 생성 방식
텍스트 본문 있음 본문 내용 + 원본 댓글 분위기 참고 → AI가 50~60대 말투로 각색
이미지 전용 (본문 텍스트 없음) 원본 댓글이 있으면 → 댓글 보고 반응 생성
원본 댓글도 없으면 → 제목만 보고 짧은 반응 생성

AI 실패 보험: Claude API가 오류가 나면 AI 필터 전체를 건너뛰고 기존 키워드 방식으로 자동 게시한다. AI 실패 때문에 글이 안 올라가는 일은 없다.

06 — AI 페르소나

5명의 AI 게시자 — 누가 글을 올리는가

스크래퍼가 올린 글은 실제 사람처럼 보이는 5명의 봇 계정이 게시자로 표시된다. 카테고리에 맞는 페르소나가 자동으로 선택된다.

07 — 펨코 예외

펨코는 왜 창업자 Mac에서만 처리되는가

펨코(fmkorea)는 Cloudflare 보안 시스템이 걸려 있다.
미국에 있는 GitHub Actions 서버에서 접속하면 자동으로 차단된다.
한국 IP + 실제 컴퓨터에서만 정상 접속이 가능하다.

08 — 시트 상태값

B열 상태로 결과 확인하기

상태값의미창업자가 할 일
PENDING 처리 대기 중 없음 (다음 실행 시간에 자동 처리)
PROCESSING 지금 처리 중 없음 (잠시 기다리면 됨)
PUBLISHED 게시 완료 F열에서 게시글 URL 확인 가능
FAILED 스크래핑 실패 G열 에러 메시지 확인 → PENDING으로 바꾸면 재시도
SKIPPED 중복 URL (이미 올라간 글) 없음 (이미 존재)
SKIPPED_LOW_SCORE AI가 관련성 낮다고 판단 G열에서 점수 확인. 억지로 올리고 싶으면 PENDING으로 바꾸면 됨
09 — 운영 정책

이것만 지키면 된다

✅ 넣으면 좋은 글
  • 50~60대 여성이 "이거 나 얘기잖아!" 할 것 같은 글
  • 유머·건강·요리·여행·가족·부부 관련
  • 공감 댓글이 많이 달린 원본 글 (HOT 탭 글)
  • 화제성 탭: 정말 반응 좋을 것 같은 글만
❌ 피해야 할 것
  • 정치·종교·혐오 관련 → AI가 필터링하지만 창업자도 주의
  • 너무 젊은 타겟 글 (10~20대 주제) → AI가 걸러냄
  • 같은 URL 두 번 입력 → SKIPPED 처리됨 (중복 방지)
  • "시니어", "어르신", "노인" 표현이 많은 글

화제성 탭 사용 기준: 파동 댓글·좋아요 비용이 더 들기 때문에 정말 좋은 글만 사용하는 것이 효율적이다. 하루 1~3개 정도가 적당하다.
