01 — 배경

왜 만들었는가

우나어는 50~60대 여성이 공감하는 콘텐츠로 채워져야 한다.
그런데 초기에는 사용자가 없으니 게시글이 없고, 게시글이 없으니 사용자가 안 온다.

03 — 창업자 역할

창업자가 하는 일 — 이게 전부다

A열에 URL만 넣으면 끝. 나머지는 전부 자동이다.
창업자가 "이 글 좋다"고 판단한 URL을 Google Sheets에 붙여넣으면, AI가 판단하고, 봇이 올리고, 파동이 HOT을 만든다.

탭이 아직 Sheet에 없어도 GHA는 실패하지 않는다.
없는 탭은 경고(warning) 로그만 남기고 자동으로 건너뛴다. 코드 먼저 배포 → 탭 나중에 추가해도 무방.

입력 방법 — 컬럼 A~J

A열에 URL만 넣으면 된다. B열 상태(Status)는 자동으로 PENDING이 된다.
나머지 열은 선택 사항이며, 비워두면 AI가 자동으로 처리한다.

J열 사용 케이스: 펨코 글이 CF_BLOCKED로 FAILED 처리됐을 때, G열에 "CF 차단 — J열에 본문 붙여넣기 가능" 메시지가 표시된다. 이때 원본 글을 복사해서 J열에 붙여넣고 B열을 PENDING으로 바꾸면 된다.

오유·네이트판·82cook은 Mac 상태와 무관하게 GitHub에서 자동 처리된다.

자동 처리 단계 (전체 흐름)

1
PENDING 상태 URL 감지
시트에서 B열 = PENDING인 행만 가져온다. 처리 시작 시 즉시 PROCESSING으로 변경 (중복 처리 방지).
중복 URL 체크: 이미 게시된 글이면 SKIPPED 처리. 파동이 없으면 파동만 재예약 후 PUBLISHED.
2
J열 확인 — 수동 모드 or 자동 스크래핑
J열(rawContent)에 내용이 있으면 → Playwright 브라우저 접속을 건너뛰고 수동 본문을 사용.
비어있으면 → Playwright로 원본 사이트에 자동 접속해 본문·이미지·댓글을 수집한다.
주의: J열 수동 모드에서는 원본 댓글 수집이 불가능하다 (sourceComments = []). 파동 댓글은 본문만으로 생성되며, 이미지 전용 글이면 제목 기반 fallback으로 대체된다.
3
Playwright 브라우저로 원본 접속 + 댓글 수집
해당 URL에 자동으로 접속 (타임아웃 45초). 페이지 로드 후 사이트별 딜레이(2~3초) 대기.
원본 댓글 최대 10개 수집 — AJAX 비동기 로딩 최대 5초 대기. 5자 초과 댓글만 수집.
수집된 댓글은 BotLog details.sourceComments에 저장, 나중에 파동 댓글 AI 각색에 활용됨.
4
콘텐츠 변환 + 이미지/비디오 R2 업로드
HTML 정제 8단계 실행 (광고 제거 → sanitize → YouTube iframe 변환 등).
이미지는 sharp로 WebP 변환 후 R2에 업로드. GIF는 원본 유지. 비디오도 R2에 업로드.
외부 도메인 차단 시에도 이미지가 우나어 서버에서 서빙됨.
5
카테고리 분류 (키워드 기반 자동 분류)
제목·본문 키워드로 카테고리를 자동 결정한다 (건강·일상·고민·자녀·유머·힐링 등).
창업자가 D열에 직접 지정했으면 그걸 최우선으로 따름. 일치 키워드 없으면 '기타'로 분류.
6
AI 관련성 판단 — 현재 비활성 (2026-05-21 제거)
Claude Haiku를 이용한 관련성 0~10점 필터는 비활성화됐다. 비용 절감(월 ~$0.68 제거) 및 창업자 큐레이션 품질에 대한 신뢰 기반 결정.
이 단계는 현재 건너뛴다. 창업자가 Sheet에 넣은 URL은 관련성 체크 없이 전부 게시된다.
7
우나어 사이트에 게시글 자동 생성
19명의 AI 페르소나 중 카테고리·게시판에 맞는 1명이 게시자로 표시 (STORY 10명 / HUMOR 5명 / LIFE2 4명).
삭제된 적 있는 게시글이면 DB row를 재활용(update), 완전 신규면 create.
게시글 URL은 제목 기반 slug로 자동 생성 — F열에 기록되는 URL이 /{id} 대신 /{slug} 형태로 표시됨 (2026-05-23 도입). 복원 글은 기존 slug 유지.
8
파동 BotLog 예약 (즉시)
화제성 탭 (isFeatured=true): WAVE_L(좋아요 +1분) + WAVE_1(공감댓글 +3분) + WAVE_2(비판댓글 +6분) + WAVE_3(역전댓글 +10분) 4개 예약.
일반 탭 (isFeatured=false): 댓글 +2분, 좋아요 +6분 2개 예약.
원본 댓글(sourceComments)과 본문 내용(rawContent)도 함께 저장.
9
시트 상태 업데이트 + Slack 알림
성공 → F열에 게시글 URL + PUBLISHED. 실패 → G열에 에러 + FAILED.
Slack 에이전트 채널에 처리 결과 전송 (게시 N건, 실패 N건, 필터 스킵 N건).
05 — AI 역할

AI가 하는 판단

2026-05-21 변경: Claude Haiku AI 품질 필터(관련성 점수·제목 최적화)가 제거됐다.
창업자가 직접 발굴한 URL의 품질을 신뢰하는 방향으로 전환. 비용 절감(월 ~$0.68)도 부수 효과.
현재 AI가 사용되는 곳은 파동 댓글 생성 (Claude Sonnet) 1곳뿐이다.

판단 1 — 관련성 점수 현재 비활성

과거에는 창업자가 URL을 넣으면 AI가 먼저 "이 글이 우리 타겟에 맞는가"를 0~10점으로 판단했다. 현재는 이 단계를 건너뛴다. Sheet에 넣은 URL은 AI 필터 없이 바로 처리된다.

이전 AI 필터 동작 방식 (참고용 — 현재 미사용) 
과거 동작:
50~60대 여성 관련성 0~10점 측정 → 6점 미만이면 SKIPPED_LOW_SCORE 처리 + G열에 사유 기록
Claude Haiku, max_tokens: 200, 건당 ~$0.0003

현재: 이 단계 제거됨. 카테고리 분류만 키워드 기반으로 실행됨.

판단 2 — 카테고리 분류 (키워드 기반)

카테고리는 키워드 매칭으로만 결정된다. AI override 없음.

  • 추천 — 추천·강추·꿀팁·정보·소개·리뷰·후기·맛집·꿀잼

AI 카테고리: 유머|일상|건강|요리|여행|기타 6개. 키워드 fallback이 더 세분화된 분류를 제공한다 (건강/일상/고민/자녀 등).

판단 3 — 제목 최적화 현재 비활성

과거에는 제목이 10자 미만이거나 특수문자만일 때 AI(Haiku)가 제목을 자동으로 개선했다. 현재는 원본 제목을 그대로 사용한다. 창업자가 C열에 제목을 직접 입력하면 그것이 최우선으로 적용된다.

판단 4 — 원본 댓글 AI 각색 (Claude Sonnet, 파동 시)

스크랩 시 원본 사이트 댓글을 최대 10개 자동 수집한다.
수집된 댓글은 파동 댓글 생성 시 AI(Sonnet)가 50~60대 말투로 각색해 활용한다.

⚠️ 트러블슈팅 — macOS TCC 전체 디스크 접근 오류

증상: Mac 재시작 후 갑자기 launchd 실행 실패 — Operation not permitted / getcwd 실패 (exit code 126)

원인: macOS TCC(전체 디스크 접근) — /bin/bash가 Full Disk Access 권한 미승인 상태. Mac 재시작 시 가끔 초기화됨.

항목내용
복구 방법시스템 설정 → 개인 정보 보호 및 보안 → 전체 디스크 접근 → /bin/bash 항목 체크박스 활성화
영향 대상naver-cafe-sheet-scraper, fmkorea-scraper 등 /bin/bash → launchd-alert.sh 래핑 구조를 사용하는 모든 launchd 잡
설계 배경scripts/launchd-alert.sh가 중간에 존재하는 이유가 바로 이 TCC 우회를 위한 것 (bash에 권한을 한 번만 주면 모든 잡에 적용됨)

콘텐츠 변환 파이프라인 8단계

스크래핑한 외부 HTML을 우나어 사이트에 맞게 변환하는 과정. agents/community/content-transformer.ts

1
불필요 요소 제거 (removeElements)
광고·네비게이션·댓글영역·iframe·서명 등 사이트별로 정의된 CSS 셀렉터로 제거. 예: .rd_hd, .botBanner, .comment_area
2
사이트별 전처리 (preClean)
펨코: link.php 리다이렉터 → 실제 URL 추출. highslide 이미지 확대 wrapper 제거. auto_media_wrapper에서 <video> 태그만 추출 (플레이어 UI 제거). button/input 태그 제거.
네이트판: /attach/imageView 뷰어 링크 wrapper 제거.
오유: <b> 태그가 본문 전체를 감싸는 패턴 제거.
3
sanitize-html 정제
허용 태그: p, br, strong, i, em, u, del, a, h2~h4, img, iframe, video, source, ul, ol, li, blockquote, hr, div, span, table
iframe은 youtube.com·youtube-nocookie.com만 허용. 나머지 외부 iframe 전부 제거.
4
후처리 (postClean) — 미디어 플레이어 UI 잔해 제거
펨코 MediaElement.js 플레이어 UI 잔해 제거: "Video Player" 텍스트, 속도 셀렉터 목록 (0.25x 등), 타임코드 (0:00 형식), 슬래시(/) 요소 등.
5
YouTube URL → iframe 변환
<a> 태그 안의 YouTube 링크 또는 텍스트 bare URL을 youtube-nocookie.com/embed iframe으로 변환. width 100%, height 400px.
6
video controls 속성 자동 주입
controls 속성 없는 <video> 태그에 자동으로 추가. 사용자가 영상을 재생할 수 있도록.
7
빈 태그 정리
빈 <p>/<div> 제거. 연속 <br> 3개 이상 → 2개로 줄임. &nbsp; → 공백 변환.
8
출처 표시 (게시판별 차이)
HUMOR (웃음방): 본문 끝에 <p>출처: 오늘의유머</p> 자동 추가.
STORY (사는이야기): 출처 없음 — 개인이 직접 쓴 글처럼 보여야 하기 때문.
LIFE2 (2막준비): 출처 없음 — STORY와 동일 정책. 인생 2막 관련 개인 경험·고민 글처럼 표시.
