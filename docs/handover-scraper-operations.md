# 외부 콘텐츠 스크래퍼 운영 가이드

> 작성일: 2026-05-15 | 대상: 창업자 (비기술 독자)
> 이 문서는 "스크래퍼가 왜 존재하고, 어떻게 움직이며, 창업자가 무엇을 해야 하는가"를 설명한다.

---

## 1. 배경 — 왜 만들었는가

우나어(우리 나이가 어때서)는 50~60대 여성이 공감하는 콘텐츠로 채워져야 한다.
그런데 초기에는 사용자가 없으니 게시글이 없다. 게시글이 없으니 사용자가 안 온다.
이 악순환을 끊기 위해 **외부 커뮤니티에서 좋은 글을 가져와 우리 사이트에 올리는 자동화 시스템**을 만들었다.

스크래퍼가 가져오는 외부 사이트는 세 곳이다:

| 사이트 | 특징 |
|--------|------|
| **오늘의유머 (오유)** | 50~60대가 많이 보는 유머·생활 커뮤니티 |
| **네이트판** | 40~60대 여성 비율 높은 커뮤니티. 연애·가족·일상 |
| **펨코 (fmkorea)** | 다양한 연령대. 유머·시사·연예 카테고리 선별 |

---

## 2. 목적 — 무엇을 위해 하는가

**단기**: 사이트가 비어 보이지 않도록 콘텐츠 채우기
**중기**: 50~60대 공감 콘텐츠가 쌓이면 → 검색 유입 → 실제 사용자 유입
**장기**: 사용자가 직접 글을 쓰기 시작하면 스크래퍼 의존도를 줄인다

스크래퍼가 올리는 글은 외부 출처가 명시되고, 우나어 스타일로 재포장된다.
복사·붙여넣기가 아니라 "우리 사이트에 어울리는 형태"로 변환해서 올린다.

---

## 3. 목표 (운영 KPI)

| 지표 | 목표 |
|------|------|
| 하루 스크래핑 처리량 | 5회 실행 × 회당 3~8건 = 하루 15~40건 처리 시도 |
| 실제 게시율 | 처리 시도의 70~80% (관련성 낮은 글 자동 필터링) |
| AI 관련성 필터 통과 기준 | 10점 만점 중 6점 이상 |
| 게시 직후 댓글 달리는 시간 | 1분 이내 첫 댓글, 10분 이내 전체 파동 완료 |

## 4. 창업자가 하는 일 — 이게 핵심이다

스크래퍼의 핵심 원리는 이것이다:

> **창업자가 "이 글 좋다"고 URL을 Google Sheets에 넣으면, 나머지는 자동으로 처리된다.**

### Google Sheets 사용 방법

**시트 주소**: (기존에 사용하던 시트)

시트에는 탭이 4개 있다:

| 탭 이름 | 어디에 올라가는가 |
|---------|----------------|
| **사는이야기** | 우나어 사는이야기 게시판 (일반) |
| **웃음방** | 우나어 웃음방 게시판 (일반) |
| **사는이야기_화제성** | 사는이야기 + 즉시 HOT 탭 올리기 |
| **웃음방_화제성** | 웃음방 + 즉시 HOT 탭 올리기 |

**화제성 탭**을 쓰면: 게시 1분 후 좋아요·댓글 파동이 즉시 투입되어 HOT 탭에 올라간다.
**일반 탭**을 쓰면: 게시는 되지만 파동 강도가 약하다.

### 입력 방법

A열에 URL만 넣으면 된다. B열 상태(Status)는 자동으로 `PENDING`이 된다.

```
A열: https://www.todayhumor.co.kr/board/view.php?table=...
B열: (비워두거나 PENDING 입력)
```

선택으로 입력 가능한 것들:

| 열 | 의미 | 예시 |
|----|------|------|
| C열 | 제목 바꾸기 | 내가 직접 제목을 정하고 싶을 때 |
| D열 | 카테고리 지정 | 유머, 건강, 일상, 요리, 여행 |
| E열 | 게시자 닉네임 | ㅋㅋ요정, 봄바람, 매일걷기 등 |

**대부분은 A열 URL만 넣으면 AI가 나머지를 알아서 판단한다.**

## 5. 자동으로 일어나는 일 — A to Z

창업자가 URL을 넣은 뒤 다음이 자동으로 일어난다.

### 5-1. 실행 시간

하루 5번, 아래 시간에 자동 실행된다:

```
07:30 KST — 아침 (전날 밤 또는 새벽에 입력한 URL 처리)
09:00 KST — 오전
12:00 KST — 점심
15:00 KST — 오후
21:00 KST — 저녁
```

예: 창업자가 오전 10시에 URL 5개를 넣으면 → 12:00 KST에 자동 처리된다.

### 5-2. 자동 처리 흐름

```
[창업자가 URL 입력 → PENDING 상태]
         ↓ (다음 실행 시간에 자동 시작)
[AI가 먼저 글을 판단]
  → 50~60대 여성과 관련 있는가? (관련성 점수 0~10점)
  → 6점 미만이면 → SKIPPED_LOW_SCORE (시트에 이유 기록, 게시 안 함)
  → 6점 이상이면 → 계속 진행
         ↓
[AI가 카테고리 분류]
  → "이 글은 유머인가, 건강인가, 일상인가?"
  → 창업자가 D열에 직접 지정했으면 그걸 따름
         ↓
[AI가 제목 확인]
  → 제목이 너무 짧거나 ㅋㅋ 같은 특수문자만 있으면 → 자동으로 더 나은 제목 생성
  → 창업자가 C열에 직접 제목을 넣었으면 그걸 따름
         ↓
[Playwright 브라우저가 자동으로 해당 URL에 접속]
  → 본문 글 추출
  → 이미지 추출 → R2(클라우드 저장소)에 자동 업로드
  → 원본 댓글 최대 10개 수집 (파동 댓글 생성 시 참고)
         ↓
[우나어 사이트에 게시글 자동 생성]
  → 게시자: 5명의 AI 페르소나 중 1명 (카테고리에 맞는 사람)
  → 출처 URL 포함
         ↓
[화제성 탭이면 → 즉시 파동 시작]
  → +1분: 좋아요 10명 자동 클릭
  → +3분: 공감 댓글 5개 자동 생성
  → +6분: 반박/다른 의견 댓글 3개 자동 생성
  → +10분: 반전·마무리 댓글 5개 자동 생성
  → 결과: HOT 탭 자동 승격
         ↓
[시트 상태 업데이트]
  → PUBLISHED (성공) 또는 FAILED (실패) 또는 SKIPPED_LOW_SCORE (AI 필터)
```

## 6. AI의 역할 — 무엇을 판단하는가

이번 개선(2026-05-15)으로 스크래퍼에 Claude AI(Haiku 모델)가 추가됐다.
**이전에는 단순 키워드 매칭**이었다면, 이제는 AI가 문맥을 이해해서 판단한다.

### AI가 하는 3가지 판단

#### 판단 1: 관련성 점수 (0~10점)

창업자가 URL을 넣어도 AI가 "이 글이 우리 타겟에 맞는가"를 먼저 본다.

| 점수 | 의미 | 처리 |
|------|------|------|
| 9~10점 | "50~60대 여성이 바로 공유할 것" | 게시 |
| 7~8점 | "우리 또래면 공감할 것" | 게시 |
| 6점 | "관련 있긴 하지만..." | 게시 (임계값) |
| 5점 이하 | "20대 타겟이거나 우리와 무관" | **SKIPPED** (시트에 이유 기록) |

예: 창업자가 실수로 20대 취업 관련 글 URL을 넣었어도 AI가 자동으로 걸러낸다.

> **주의**: AI도 실수할 수 있다. 좋은 글인데 SKIPPED됐다면 → D열에 카테고리를 직접 지정하고 B열을 다시 PENDING으로 바꾸면 재처리된다.

#### 판단 2: 카테고리 분류

키워드 매칭의 문제: "남편이 웃음을 잃었어요" → 웃음방(HUMOR)으로 잘못 분류
AI 분류: 문맥을 보고 → 사는이야기(고민) 게시판으로 정확하게 분류

| AI 분류 카테고리 |
|----------------|
| 유머, 일상, 건강, 요리, 여행, 기타 |

#### 판단 3: 제목 최적화

원본 제목이 "ㅋㅋㅋ" 또는 "이거 보세요" 수준일 때만 개선안을 만든다.
원본 제목이 충분하면 그대로 유지한다.

예: "ㅋㅋㅋ 이거 실화냐" → "남편 자신감이 너무 넘쳐서 웃음 터진 일"

> **AI 실패 보험**: Claude API가 오류나면 AI 필터 전체를 건너뛰고 기존 키워드 방식으로 자동 게시한다. AI 실패 때문에 글이 안 올라가는 일은 없다.

## 7. 5명의 AI 페르소나 — 누가 글을 올리는가

스크래퍼가 올린 글은 실제 사람처럼 보이는 5명의 봇 계정이 게시자로 표시된다.
카테고리에 맞는 페르소나가 자동으로 선택된다.

| 코드 | 닉네임 | 성격 | 주로 올리는 글 |
|------|--------|------|--------------|
| C | **ㅋㅋ요정** | 웃음 포인트에 반응 빠름 | 유머·웃긴 이야기 |
| E | **봄바람** | 따뜻하고 공감 잘 함 | 일상·고민 이야기 |
| H | **매일걷기** | 건강에 관심 많음 | 건강·운동 이야기 |
| I | **한페이지** | 감성적·차분함 | 힐링·감동 이야기 |
| P | **오후세시** | 만능형. 추천·자랑·자녀 이야기 좋아함 | 기타 모든 분야 |

이 5명은 게시글 올리기 + 화제성 파동 댓글 쓰기 모두 담당한다.

## 8. 펨코는 왜 다르게 처리되는가

펨코(fmkorea)는 **Cloudflare 보안 시스템**이 걸려 있어서 해외 서버(GitHub Actions 미국 서버)에서 접속하면 차단된다. 한국 IP + 실제 컴퓨터에서만 접속 가능하다.

그래서:
- 오유·네이트판 → **GitHub Actions(클라우드)** 에서 처리
- 펨코 → **창업자 Mac 컴퓨터** 에서 처리 (launchd 자동 실행)

| 항목 | 오유·네이트판 | 펨코 |
|------|-------------|------|
| 실행 서버 | GitHub 클라우드 | 창업자 Mac |
| 실행 시간 | 07:30·09:00·12:00·15:00·21:00 KST | 11:30·21:30 KST |
| 자동화 여부 | 완전 자동 | Mac이 켜져 있어야 실행됨 |

**주의**: 펨코 처리는 창업자 Mac이 해당 시간에 켜져 있고 인터넷 연결이 돼야 한다.

## 9. 시트 상태값 — 결과를 어떻게 확인하는가

시트의 B열(Status)이 처리 결과를 보여준다:

| 상태 | 의미 | 창업자가 할 일 |
|------|------|--------------|
| **PENDING** | 처리 대기 중 | 없음 (다음 실행 시간에 자동 처리) |
| **PROCESSING** | 지금 처리 중 | 없음 |
| **PUBLISHED** | 게시 완료 | F열에 게시글 URL 확인 가능 |
| **FAILED** | 스크래핑 실패 | G열 에러 메시지 확인. PENDING으로 바꾸면 재시도 |
| **SKIPPED** | 중복 URL (이미 존재) | 없음 |
| **SKIPPED_LOW_SCORE** | AI가 관련성 낮다고 판단 | G열에 점수 확인. 억지로 올리고 싶으면 PENDING으로 바꾸면 됨 |

## 10. 운영 정책 — 이것만 지키면 된다

### 하면 좋은 것
- 50~60대 여성이 "이거 나 얘기잖아!" 하고 공유할 것 같은 글
- 유머·건강·요리·여행·가족·부부 관련 글
- 공감 댓글이 많이 달린 원본 글 (HOT 탭에 있는 글)

### 하면 안 되는 것
- 정치·종교·혐오 관련 글 → AI가 필터링하지만 창업자도 주의
- 너무 젊은 타겟의 글 (10~20대 주제) → AI가 걸러냄
- 같은 URL을 두 번 넣기 → SKIPPED 처리됨 (중복 방지)

### 하루에 몇 개가 적당한가
- 딱 정해진 제한은 없다
- 창업자가 직접 선별하는 것이 핵심. 양보다 질
- 화제성 탭은 정말 좋은 글만: 파동 비용이 더 들기 때문

## 13. 창업자가 기억할 3가지

```
1. A열에 URL 넣으면 끝. 나머지는 자동.
   (다음 정각 실행 시간에 알아서 처리됨)

2. B열 상태가 SKIPPED_LOW_SCORE면 → AI가 관련성 낮다고 판단
   (억지로 올리고 싶으면 PENDING으로 바꾸면 됨)

3. 펨코 글은 Mac이 켜져 있어야 처리됨
   (오유·네이트판은 Mac 꺼져 있어도 됨)
```
