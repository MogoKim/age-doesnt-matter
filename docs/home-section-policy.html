💬
커뮤니티 3섹션 (뜨는이야기·사는이야기·웃음방) 기본 선발은 24시간 이내 + 좋아요/댓글 1개 이상. 반응이 없으면 기본 선발에서 제외되지만, 해당 게시판 후보가 0개인 경우 7일 fallback(반응 조건 없음)으로 노출될 수 있습니다.

매거진 · 오늘의 추천 일자리 반응과 무관하게 발행 후 최대 60초 내 홈 쿼리에 반영됩니다. 최신 N개(매거진 4개 · 일자리 5개) 안에 들면 홈에 노출됩니다.
📌
어드민 홈 큐레이션 — 이제 직접 편성 가능 (신규) 어드민이 뜨는이야기·사는이야기·웃음방 섹션에 글을 고정(PIN)/숨김(HIDE)할 수 있습니다(만료 4종). 단, 이와 별개인 "집중"(featured) 체크는 여전히 홈 노출과 무관합니다.
📖
글상세 관련글 추천 v2 — 내부 순환 추천 (신규) 홈 알고리즘이 "첫 화면 노출 정책"이라면, 관련글 추천 v2는 "글상세 내부 순환 추천 정책"입니다. A/B 실험(exp1_related_flow) 종료 후 B 방향 채택 → 이제 항상 📖 다음에 읽기 좋은 이야기 카드 3개를 노출합니다. 추천 기준은 맥락 × 흥미도. 현재 글·이번 세션에서 이미 본 글·가입인사 글은 제외합니다. 추가 DB 쿼리 없이 서버가 후보 24개를 가져와 브라우저(클라이언트)에서 점수화합니다.

뜨는점수란?

⚡
즉시 계산 (실시간)
좋아요·댓글·비회원 댓글이 달리는 순간 바로 갱신
🔄
배치 재계산 (하루 2회)
GHA 자동 실행 · KST 12:11 · 18:11
📐 화면 선발용 점수 vs DB 저장 점수 — 두 가지가 별도로 존재
1단계 — 후보 풀 조회 (DB trendingScore 기준)
getHomeBoardHotPostsRaw()가 DB trendingScore 내림차순으로 게시판별 후보 풀을 가져옴 (STORY 12개 / LIFE2 4개 / HUMOR 4개).
2단계 — currentScore 계산 (DB 미저장)
getTrendingQuotaPosts()가 후보 풀 내 각 글의 현재 likeCount·commentCount·viewCount·작성시간으로 즉시 재계산. 이 값은 DB에 저장하지 않음.
3단계 — 쿼터 적용 + 통합 정렬
게시판별 상위 쿼터(STORY 6 / LIFE2 2 / HUMOR 2)로 자른 뒤, 통합 currentScore 내림차순으로 최종 정렬 → 최대 10개 표시.
DB trendingScore 갱신 경로: ① engagement 발생 시 즉시 갱신 (refreshPostTrendingScore) ② COO 배치 에이전트 하루 2회 전체 재계산 (KST 12:11·18:11). 1단계 후보 정렬에 사용됨.
뜨는점수 = (댓글×5 + 좋아요×3 + 조회수×0.1)
               × 시간대보너스
              ÷ (글 작성 후 경과시간 + 2)^1.5
이 공식 한 줄 요약
📈 반응이 많을수록 점수 올라감 (댓글 5점 > 좋아요 3점 > 조회수 0.1점)
📉 글을 올린 후 시간이 지날수록 점수 빠르게 내려감
🌙 새벽·오전에 달린 반응은 더 높은 가치로 인정 (최대 ×1.5)

🔁 2단 구조 — 서버 후보 조회 → 브라우저 점수화
1단계 — 서버 후보 24개 (캐시)
① 같은 게시판 + 같은 category 우선 → ② 부족하면 같은 게시판 fallback. 정렬 trendingScore desc, createdAt desc. 캐시 revalidate: 300(5분, 사용자 무관이라 안전).
2단계 — 브라우저 점수화 + 제외
받은 24개에서 현재 글·이미 본 글·가입인사 제외 후 맥락 × 흥미도 점수 계산 → 상위 3개만 표시. 이 계산은 폰 안에서 일어나 추가 서버 요청 0.

추천 이유(reason) 라벨 — 각 카드에 "왜 추천했는지" 표시

내부 코드화면 라벨언제 붙나
topic같은 주제같은 category + 제목/요약 키워드 겹침이 가장 큼
comment댓글 많은 글댓글 수 기여가 가장 큼
like많이 공감한 글좋아요 수 기여가 가장 큼
trending요즘 많이 보는 글trendingScore/조회수 기여가 가장 큼
recent최근 이야기최신성 기여가 가장 큼

점수 가중치 (REC_WEIGHTS) — 맥락 점수 + 흥미도 점수를 섞는다

💡 맥락만 맞는 재미없는 글도, 인기만 높은 엉뚱한 글도 피하기 위해 맥락 점수와 흥미도 점수를 섞는다.
각 후보 글에 아래 점수를 더해 합산하고, 높은 3개를 뽑는다. 흥미도 항목은 후보 24개 안에서의 상대 비율로 정규화한다(가장 높은 글이 만점).
구분항목최대 점수쉽게
맥락sameCategory+30지금 글과 같은 주제(카테고리)
titleOverlap+20제목에 같은 단어가 겹칠수록
previewOverlap+10요약(미리보기)에 같은 단어
흥미도trending+15요즘 뜨는 글(trendingScore)
comment+12댓글 많은 글
like+8공감(좋아요) 많은 글
view+5조회수 높은 글
recency7+87일 이내 최신 글
recency30+430일 이내 글
💡 점수 기준(+30, +20…)은 REC_WEIGHTS 상수 한 곳에서 관리합니다. "댓글 많은 글을 더 밀자" 같은 조정은 숫자만 바꾸면 됩니다(코드 구조 변경 불필요). 롤백도 숫자만 되돌리면 됩니다.

관찰 지표

  • 세션PV — 추천 도입 전후 비교
  • 3화면 도달률 — 한 세션에 글 3개+ 읽는 비율
  • 현재글/이미 본 글이 추천에 안 나오는지 샘플 확인
