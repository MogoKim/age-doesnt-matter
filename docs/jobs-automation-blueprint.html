왜 일자리인가 (전략)

우리 또래(50·60대)에게 "일자리"는 가장 절실한 관심사 중 하나입니다. 재취업·부수입·소일거리 — 돈과 직결된 욕망(MONEY)이라 검색도 많이 하고, 자주 들어옵니다.

축효과
수익일자리 페이지에 AdSense 광고 + 쿠팡 배너 노출 → 광고 수익
유입"OO지역 일자리"로 검색하고 들어온 신규 방문자 → 회원가입 유도
SEO매일 새 공고 = 매일 새 페이지 → 검색엔진이 좋아하는 "신선한 콘텐츠"

기능 관리 대장 기준 F06(일자리 게시판)·A03(일자리봇)·A17(일자리 매처)로 등록돼 있으며, 충족 욕망은 MONEY, 타겟은 P4 페르소나입니다.

어떻게 고르나 (선별)

50건을 다 올리지 않습니다. 품질 등급(Tier) + 균형 정책(쿼터)으로 매회 최대 8건만 엄선합니다.

AI가 하는 일 (가공)

긁어온 공고는 딱딱하고 어렵습니다. AI(Claude Sonnet)가 우리 또래 눈높이로 다시 씁니다. 단, 비용 아끼려고 꼭 필요한 4가지만 AI에 맡깁니다.

AI가 만드는 것설명
제목·부제"[서울 강남구] 도서관 사서 보조" + "오전만 근무, 오후는 내 시간"
SEO 키워드 4개검색 노출용 숨은 키워드
Pick 포인트 5개"주말 완전 휴무 🗓️", "앉아서 하는 업무 💺" 등 매력 요약
Q&A 3개"무릎이 안 좋은데 오래 서있어야 하나요?" 같은 실제 궁금증

AI를 안 쓰는 부분 (규칙으로 처리 → 비용 절감)

급여 정규화표시 태그(나이무관 등)
항목값
사용 모델claude-sonnet-4-6 (고객이 보는 콘텐츠라 고품질)
호출 횟수공고 1건당 4번 × 8건 × 3회/일 = 약 96번/일 추정
실패 대비AI 결과가 불량(원본 복사·길이 초과)이면 규칙 기반 Fallback으로 자동 대체
품질 가드"시니어 환영", "나이 무관" 같은 뻔한 표현은 프롬프트에서 명시적으로 금지 → 공고마다 구체적이고 다른 내용이 나오게 합니다.

검색 노출 (SEO)

단순히 올리는 게 아니라 구글·네이버가 잘 찾아가도록 설계돼 있습니다. 특히 구글 일자리 검색(Google for Jobs)에 노출될 수 있습니다.

장치역할
JobPosting구글이 "이건 채용공고"라고 인식하는 구조화 데이터 → 구글 일자리 검색 노출
BreadcrumbList홈 > 일자리 > 상세 경로 표시
ItemList지역 페이지의 공고 목록을 구조화
sitemap일자리 목록·상세 전부 + 지역 17개를 검색엔진에 자동 제출 (우선순위 0.9/0.7)
OG 이미지공고별 카카오/SNS 공유 시 썸네일 자동 생성
신선도가 핵심매일 새 공고가 올라가야 구글 일자리 검색이 계속 노출해 줍니다. (그래서 자동화 중단이 SEO에 치명적 — 10번 섹션 참고)

모니터링 & 리스크

매 실행 결과는 슬랙으로 자동 보고됩니다. 다만 "성공이라고 떴는데 실제론 안 된" 사고가 있었습니다.

⚠️ 과거 사고 — Silent Failure (2026-05-16 ~ 06-03, 약 3주) 코드 구조 변경(process.exit 추가) 때문에, 일자리봇이 "시작합니다"라고 대답만 하고 실제 크롤링 전에 프로세스가 종료됨. GitHub Actions에는 "성공"으로 표시돼서 3주간 아무도 몰랐고, 그동안 신규 공고 0건 → 검색 노출도 하락.

→ 원인: 봇이 백그라운드로 일하는데 관리자가 즉시 종료 / 복구: 끝까지 기다리도록 수정(main() await), 즉시 정상화 확인.
