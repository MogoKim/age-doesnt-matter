01

목적과 배경 — 왜 바꾸는가?

현재 문제

게시글이 너무 적다

하루 25~30건 발행. 09~23시 사이 시간당 2건꼴. 방문자가 들어와도 새 글이 없어 보여 커뮤니티가 죽어있는 것처럼 느껴진다.

현재 문제

AI가 쓴 글 티가 난다

일반글은 카페 트렌드 주제만 힌트로 받아 AI가 "완전 창작" 방식으로 씀. 실제 50·60대 커뮤니티 특유의 표현과 맥락이 없어 어색하다.

현재 문제

크롤링 기준이 너무 높다

품질 점수 60점 이상만 "발행 가능" → 실제 수집 글의 83.2%가 탈락. 좋은 글 재료가 버려지고 있다.

현재 문제

게시판 분배 불균형

사는이야기에 글이 집중되고 2막준비·웃음방은 상대적으로 빈약. 방문자가 원하는 콘텐츠를 찾기 어렵다.

기획 핵심 방향:
네이버 카페에서 실제로 오가는 말과 감정을 90% 이상 그대로 가져와 자연스러운 글을 만들고, 09시~23시 매 시간 5건씩 꾸준히 발행해 커뮤니티가 살아있다는 느낌을 준다. AI 창작 비율을 최소화하고, 실제 카페 커뮤니티의 목소리를 우나어로 옮기는 것이 목표.

왜 이게 핵심인가?
75건/일 발행을 위해서는 충분한 원문 재료가 있어야 한다. 현재 기준으로는 160건 수집 중 24건만 사용 가능 → 재료 부족. 기준을 완화해 약 64건 이상이 발행 가능 상태가 되면 시간당 5건 발행에 충분한 재료 확보.

원문 보존율 강화 — "자연스러운 커뮤니티 글" 만들기

일반글 (Track 1)

원문 90% 유지 방식으로 전환

현재: 카페 트렌드 주제만 힌트 → AI가 처음부터 창작 → 어색함

V2: 실제 카페글 원문을 90% 그대로 가져오고, 10%만 변형
변형 항목: 지명·인명 일부 변경 + 어미 1~2개 + 페르소나 도입부 1~2문장 추가

효과: 실제 50·60대가 쓴 말투·감정·표현이 그대로 살아있어 AI 티 최소화

화제성글 (Track 2)

원문 95% 유지 — 현행 방식 유지

이미 95% 원문 보존 방식으로 운영 중.
갈등·감정흐름·금액 보존 + 지명/인명 익명화 + 어미 1~2개 변형.

V2에서 변화 없음. 현재 방식이 잘 작동 중.

댓글 · 대댓글

원문 댓글 90% 유지로 전환

현재: AI가 새로 생성 → 표준어·정중한 표현 위주 → 티남

V2: 크롤링 시 수집한 실제 댓글을 90% 원문 보존
변형: 어미 또는 감탄사 1개만 변경 (킬러글 방식 동일 적용)

전제: 크롤링 시 댓글 최소 5~10개 이상 수집 필요 (DEEP 모드 기준 현재 10개 수집 중)

페르소나 구조 개편 — 글 전문 vs 댓글 전문 분리

글 전문 페르소나 (포스터)
  • 각 게시판별 고정 담당 페르소나
  • 사는이야기 전담 / 2막준비 전담 / 웃음방 전담
  • 시간대별 발행 스케줄에 배정
  • 글만 쓰고 댓글은 최소 (자기 글에만 1~2개)
  • 최소 35명 이상 확보 목표
댓글 전문 페르소나 (리액터)
  • 글은 안 쓰고 댓글·대댓글만 전담
  • 공감형 / 질문형 / 반론형 / 유머형 유형 구분
  • 5분 이내 파동 첫 타자 역할
  • 최소 15명 이상 확보 목표
  • 여러 글에 동시에 배치 가능 (글 쓸 필요 없음)
구조 정리: 현재 50명 페르소나를 역할에 따라 재분류 → 글 전문 35명 + 댓글 전문 15명. 추가 페르소나가 필요하면 댓글 전문 위주로 신설 (글 쓰는 능력 불필요, 댓글 1~2문장 특화만 있으면 됨).

큐레이션 전면 통합 — 욕망 외 콘텐츠도 수집·발행

현재 문제
  • 큐레이션은 DEEP(08:30)에서만 3건 발행
  • 욕망 카테고리(HEALTH, FAMILY 등 13가지)로만 필터
  • 욕망 분류가 안 되는 일상·유머·정보글 탈락
  • 하루 3건은 너무 적음
V2 목표
  • DEEP + QUICK + CRAWL 3회 모두 큐레이션 포함
  • 욕망 분류 외에 "유머", "정보", "일상" 카테고리도 수집
  • 게시판별 비율에 맞춰 큐레이션 글도 배분
  • 큐레이션도 원문 90% 보존 방식 적용
핵심 변경: 큐레이션 글도 일반글 75건 발행 풀(Pool)에 포함해서 함께 관리. 별도 3건 고정 발행이 아니라, 시간대별 5건 중 일부를 큐레이션 방식으로 채우는 구조로 통합.

개발 우선순위 — 무엇부터 시작할 것인가?

P0
크롤링 품질 기준 완화 (isUsable 60→40점)
재료가 늘어야 나머지 모든 것이 가능. 1줄 수정으로 즉시 효과. config.ts QUALITY_THRESHOLDS 수정.
P0
GHA 스케줄 09~23시 매 시간 고정
현재 불규칙 13회 → 매 시간 정각 15회로 변경. agents-seed.yml cron 수정.
P1
scheduler.ts 게시판 비율 테이블 추가
시간당 5건 × 사는이야기 50% · 2막준비 35% · 웃음방 15% 자동 배분 로직.
P1
일반글 원문 90% 보존 방식으로 전환
generator.ts의 generatePost() 프롬프트 변경. 현재 "창작"에서 "원문 90% 유지 + 10% 변형"으로.
P2
댓글 파동 강화 (5분 이내 4개 / 화제성 8개)
scheduler.ts 파동 타이밍 수정. 댓글 전문 페르소나 역할 정의 추가.
P2
댓글·대댓글 원문 90% 보존
generateComment() / generateReply() 프롬프트를 킬러글 방식과 동일하게 변경.
P3
페르소나 역할 재분류 (글 전문 / 댓글 전문)
기존 50명 페르소나를 포스터 35명 + 리액터 15명으로 역할 분류. 추가 필요 시 리액터 신설.
P3
큐레이션 전면 통합 + 서브카테고리 체계 반영
content-curator.ts maxPosts 조정. 욕망 외 "유머·정보·일상" 카테고리 추가.
