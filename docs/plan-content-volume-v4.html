02

전체글보기 실증 분석 ✓ Playwright 직접 확인

cafe.naver.com/f-e/cafes/{cafeId}/menus/0
✅ 확인됨

게시판명 정확히 포함

각 글 항목에 게시판 링크가 완전한 이름으로 포함됨. 잘리지 않음.

  • "자유 주제" → /menus/34
  • "갱년기 몸 증상" → /menus/8
  • "딸아들 이야기" → /menus/5
  • "가입인사" → /menus/43
✅ 확인됨

articleId 순차 증가

글 ID가 순차적으로 증가하는 정수. DB에 수집된 최대 ID 저장 후 비교 → 중복 방지가 단순해짐.

420293 → 420292 → 420290 → ...

✅ 확인됨

로그인 없이 목록 열람 가능

전체글보기 목록은 비로그인 상태에서도 완전히 렌더링됨. 본문 진입 시 로그인 필요 여부는 별도 확인 필요.

⚠ 처리 필요

공지/필독 상단 고정

필독 2개 + 공지 6개 = 상단 8개 고정. 게시판명 셀에 "공지"/"필독" 텍스트가 있어 CSS selector로 쉽게 필터링 가능.

⚠ 주의

좋아요수 목록에 없음

목록에는 조회수·댓글수만 표시. 좋아요수(likeCount)는 본문 진입 시에만 수집 가능.

대응: 화제성 기준을 commentCount ≥ 5로 조정. DEEP 크롤 시 본문 진입하면 likeCount도 함께 수집.

ℹ 참고

페이지당 15개 (UI 고정)

현재 UI에서 15개씩 표시. 크롤러는 페이지를 넘기며 원하는 수만큼 수집 가능. 페이지네이션은 버튼 방식(JavaScript).

URL 구조 (확인됨):
목록: https://cafe.naver.com/f-e/cafes/{cafeId}/menus/0?viewType=L
개별 글: https://cafe.naver.com/f-e/cafes/{cafeId}/articles/{articleId}?boardtype=L&referrerAllArticles=true
03

새 크롤링 아키텍처

config.ts 게시판 설정 전면 교체

Before (V3 이하) — 게시판별 순회

CAFE_CONFIGS[0].boards = [
  { name: '갱년기 몸 증상', menuId: 8, maxPages: 3, priority: 'high' },
  { name: '갱년기 마음 증상', menuId: 9, maxPages: 2, priority: 'high' },
  { name: '혼잣말 반말일기', menuId: 12, maxPages: 1, priority: 'medium' },
  ... 총 30~40개 게시판 반복
]

// → 게시판마다 URL 순회, 설정 관리 복잡, 새 게시판 추가 누락 위험

After (V4) — 전체글보기 단일 URL

CAFE_CONFIGS[0] = {
  cafeId: '29349320',
  cafeName: '우아한갱년기',
  allArticlesUrl: '/f-e/cafes/29349320/menus/0',  // 전체글보기
  maxArticlesPerCrawl: 200,         // 한 번에 수집할 글 수
  skipIfArticleIdExists: true,    // articleId 중복 방지
  filterNotice: true,            // 공지/필독 자동 제거
}

// → 게시판 설정 불필요, 신규 게시판 자동 포함, 유지보수 0
articleId 중복 방지 로직: 크롤러가 글 목록을 순회하다가 DB에 이미 존재하는 articleId를 만나면 이전 회차에서 수집 완료된 지점으로 간주하고 중단. 순차 증가 구조이므로 전체 DB 조회 없이 MAX(articleId) 하나만 비교하면 됨.
04

크롤 스케줄 — 4회로 확정

01:30 새벽 크롤 불필요 (전체글로 자동 커버)
왜 01:30이 필요 없나: 08:30에 전체글보기를 크롤하면 00:00~08:30 사이에 올라온 새벽 글이 전부 포함된다. 게시판별 순회에서는 페이지 수 제한으로 누락될 수 있었지만, 전체글보기는 최신순 통합 목록이라 새벽 글이 그대로 보인다.
회차시각 (KST)모드실행 내용수집 대상
1회차 08:30 DEEP 전체글보기 크롤 (댓글 포함) 전날 21:30 이후 ~ 당일 08:30 글 (새벽글 포함)
2회차 11:30 DEEP + 분석 전체글보기 크롤 → 심리분석 → 트렌드 → DailyBrief → 큐레이션 08:30 이후 신규 글 + 인게이지먼트 쌓인 오전 글
3회차 15:30 DEEP 전체글보기 크롤 → 큐레이션 11:30 이후 오후 신규 글
4회차 21:30 DEEP 전체글보기 크롤 → 큐레이션 저녁 신규 글 + 내일 아침 분석 인풋 확보
심리분석·트렌드·DailyBrief는 11:30에만: 하루 방향을 오전 글 인게이지먼트가 충분히 쌓인 후(11:30) 분석하여 정확도 확보. 비용 효율 유지. 나머지 3회는 크롤 + 큐레이션만.

화제성글 구분 기준 — commentCount 기준 상향

화제성글 (KILLER)

선별 기준 (V4 조정)

  • viralType NOT NULL
  • commentCount ≥ 5 (V3의 ≥2에서 상향 — likeCount 목록 미노출 보완)
  • qualityScore ≥ 50
  • 최근 7일 이내
  • 본문 150자 이상
  • DEEP 크롤 후 likeCount도 추가 확인
일반글 (REGULAR)

선별 기준 (V3 유지)

  • isUsable = true (qualityScore ≥ 40)
  • 화제성 조건 미충족
  • 최근 14일 이내
  • desireCategory NOT NULL
