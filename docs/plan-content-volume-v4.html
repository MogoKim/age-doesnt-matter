01

목적 & 배경 — 왜 V4가 필요한가

현재 시스템의 핵심 문제 4가지: 지금 운영 중인 시스템은 하루 25~30건을 발행하지만, 콘텐츠 품질과 인게이지먼트 모두 목표치에 미치지 못한다. 이를 해결하지 않으면 커뮤니티 신뢰도가 저하되고 자동화 효과가 반감된다.
문제 ①

발행량 부족 + 인풋 낭비

현재 게시판 30~40개를 하나하나 돌며 크롤한다. 설정 유지 비용이 크고, 새 게시판이 생기면 자동 누락된다. isUsable 기준(60점)을 통과하는 글이 14.6%뿐이라 하루 발행 인풋이 턱없이 부족하다. 목표 75건/일 달성 불가.

문제 ②

AI 티나는 콘텐츠

현재 '반반이 방식': 원문 50% + 페르소나 경험 50% 합성. 실제 카페 글과 너무 달라 인위적으로 보이고, 독자들이 위화감을 느껴 이탈한다. 커뮤니티 신뢰도 저하의 직접 원인.

문제 ③

초기 인게이지먼트 없음

글 올린 후 첫 댓글이 10분 후. 네이버·우나어 알고리즘은 게시 직후 반응 속도에 민감하다. 1분 내 반응이 없으면 '활성 글'로 분류되지 않아 추천에 걸리지 않는다.

문제 ④

서브카테고리 랜덤 배정

현재 categoryMap에서 무작위 선택. 게시글의 실제 내용과 카테고리가 불일치하는 경우 발생. UX 저하 + 검색 정확도 감소. DB에 서브카테고리 메타데이터도 없어 통계 불가.

문제 ⑤

아침 크롤 타이밍 미스매치

08:30 크롤은 00~08:30 새벽 글을 수집한다. 이 글들은 인게이지먼트가 거의 0에 가깝다. 화제성 기준(commentCount ≥ 2)을 통과하는 글이 극소수 → 화제성 발행 풀이 텅 빈다.

V4 목표

V4로 해결하는 것

  • 전체글보기 단일 크롤 → 설정 단순화 + 새벽글 자동 커버
  • isUsable 40으로 완화 → 발행 인풋 2.7배
  • 원문 90% 보존 → AI 티 제거
  • 댓글 파동 1분 이내 첫 반응
  • desireCategory 기반 서브카테고리 자동 배정
  • 11:30 크롤로 타이밍 최적화

03

전체글보기 크롤 아키텍처 ✓ Playwright 실증

검증 완료: https://cafe.naver.com/f-e/cafes/{cafeId}/menus/0?viewType=L URL에서 전체 게시판 글이 최신순 통합 제공됨. 게시판명·articleId·댓글수·조회수 모두 확인. 비로그인 상태에서 목록 열람 가능.

왜 비로그인 열람이 중요한가

크롤러 안정성 직결: 네이버 로그인은 2FA·보안 정책이 복잡해 자동화가 어렵고 세션이 수시로 만료된다. 로그인 없이 목록 접근이 가능하다는 것은 크롤러가 인증 관리 없이 24/7 무중단 실행 가능함을 의미한다. 세션 만료로 인한 크롤 실패 리스크 0.

기존 → V4 코드 변화

// Before: 게시판 40개 URL 순회
for (const board of CAFE_CONFIGS[0].boards) {
  await crawlBoard(board.menuId, board.maxPages)
}

// After: 전체글보기 단일 URL
await crawlAllArticles(cafeId, { maxArticles: 200, stopOnExisting: true })

공지 처리 정책

별도 필터 로직 불필요. 공지·필독 글은 본문 내용이 이벤트·홍보 성격이라 qualityScore가 낮게 산출되어 isUsable(30점) 기준에서 자연 탈락한다. 다만 제목에 "[공지]", "필독", "이벤트 당첨" 패턴이 있으면 수집 자체를 스킵하는 최소한의 문자열 필터는 두어 불필요한 본문 진입을 방지한다.
04

중복 수집 방지 정책 — 상세

원칙 1 — articleId 비교

이미 수집한 글 스킵

크롤 시작 전 DB에서 해당 카페의 MAX(articleId)를 조회한다. 목록 순회 중 이 값 이하의 articleId를 만나면 이후 모든 글은 이미 수집됨으로 판단하고 중단.

⚠ 주의: 글 삭제 시 articleId가 불연속일 수 있음. 연속 3개 이상 이미 있으면 중단.

원칙 2 — 수집 시간 창

최근 2일 이내만 수집

작성일이 2일 초과한 글은 수집하지 않는다. 오래된 글은 인게이지먼트가 고정되어 화제성 선별 의미가 없고, DB 용량 낭비.

⚠ 날짜 오염 필터: 제목·본문에 특정 날짜(예: "23일", "일요일")가 명시된 글은 오늘 날짜와 다를 경우 발행 스킵. 문맥상 시제 불일치 방지.

원칙 3 — 재크롤 갱신

같은 글, 데이터 갱신

이미 DB에 있는 글이라도 수집 후 7일 이내라면 likeCount·commentCount·topComments를 갱신한다. 인게이지먼트가 시간에 따라 변하기 때문. 8일 이후는 고정값 유지.

원칙 4 — 발행 중복 방지

같은 글 두 번 발행 금지

content-curator가 CafePost를 선택할 때 usedAt IS NULL 조건 필수. 발행 완료 시 usedAt = NOW() 기록. 동일 원문이 중복 발행되는 것 방지.

06

일반글 vs 화제성글 — 정확한 정의

일반글 REGULAR

커뮤니티 활성화 콘텐츠

목적: 게시판에 글이 꾸준히 올라오게 만들어 커뮤니티 활성도 유지. 자연스러운 일상 대화처럼 보여야 함.

선별 기준 (AND 조건):

  • isUsable = true (qualityScore ≥ 30)
  • desireCategory NOT NULL
  • 최근 2일 이내 게시
  • 본문 50자 이상
  • 화제성 조건 미충족
  • 제목·본문 날짜 오염 없음

발행: 75건/일 중 대부분. 09~23시 매 시간 배분.

화제성글 KILLER

강한 반응 유도 콘텐츠

목적: 댓글·좋아요를 폭발적으로 끌어내어 플랫폼 알고리즘 활성화. 갈등·공감·반전이 있는 글이어야 함.

선별 기준 (AND 조건):

  • viralType NOT NULL (BETRAYAL/INJUSTICE/CONTROVERSY/REVERSAL/EMPATHY)
  • commentCount ≥ 5 (좋아요 목록 미노출 보완)
  • qualityScore ≥ 50
  • 최근 7일 이내
  • 본문 150자 이상
  • DEEP 크롤 후 likeCount도 추가 확인

발행: 큐레이션 경로. 최대 12건/일 (4회 × 3건). prime time 우선 배정.

제목 처리 정책 — 약간 변형 허용

유형제목 변형 방향원칙예시
일반글 SEO 키워드 1개 추가 또는 구어체 → 검색어 형태로 정리 원문 핵심 단어 반드시 유지. 검색에 유리한 형태로만 조정. "갱년기인지 우울증인지 모르겠어요" → "갱년기 우울증 증상, 구분하는 방법 아시나요"

원문 보존 — 수미상관 방식

일반글 — 90%

구조

  1. 제목: 약간 변형 (SEO)
  2. 첫 문장: 페르소나 공감 1줄
  3. 본문: 원문 그대로 90%
  4. 마지막 문장: 페르소나 질문·여운 1줄

"저도 비슷한 상황이라 공유해요.
[원문 그대로]
혹시 다른 분들은 어떻게 하셨어요?"

화제성글 — 95%

구조

  1. 제목: 갈등 포인트 강조 변형
  2. 첫 문장: 없거나 1줄 도입
  3. 본문: 원문 95% 그대로
  4. 마지막: 변경 없음

화제성글은 원문 자체가 강력한 콘텐츠. 수정 최소화가 반응 극대화.

댓글 — 90%

구조

  1. topComments 원문 사용
  2. 어미·말투만 변환 (~해요 → ~하더라고요)
  3. 고유명사·수치 100% 유지
  4. 글쓴이 페르소나 제외 필수

실제 카페 반응 그대로 → 공감도 자연스러움.

게시판 & 서비스 카테고리 (서브카테고리) 재설계

기존 게시판 유지 + 서브카테고리 desireCategory 기반 자동 배정
현재 문제: categoryMap에서 무작위 선택. 게시글 내용과 카테고리 불일치 빈번. DB에 서브카테고리 메타데이터 없어 통계·필터 불가.
V4 해결: 심리분석에서 태깅된 desireCategory를 게시판별 서브카테고리로 자동 매핑. 누락 없이 정확하게 배정.
베스트 게시판: promotionLevel = 'HALL_OF_FAME'(좋아요 50+) 또는 'HOT'(좋아요 10+) 글을 자동 노출하는 큐레이션 공간. 시드봇이 직접 발행하는 게시판이 아님. 설계 변경 없음.
