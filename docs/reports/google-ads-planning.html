01 이미지 올리는 방법 — Playwright? API?

🚫
Playwright(UI 자동화) 사용 안 합니다 Google이 자동화 봇을 감지해서 계정 정지 리스크가 있음. 처음부터 공식 API로 설계됨.
✅
공식 Google Ads API 사용 (google-ads-api npm 패키지) PNG 파일 → base64 인코딩 → Google Ads 이미지 에셋으로 업로드. 봇 감지 없음, 안전함.

전체 자동화 흐름

단계파일하는 일방법
이미지 업로드 upload-creatives.ts PNG 16장 → Google Ads 이미지 에셋 등록 API (base64)
캠페인 생성 create-campaigns.ts 3개 캠페인 + 6개 광고그룹 + 키워드 + RSA 광고 일괄 생성 API
일일 리포트 daily-report.ts 매일 09:00 KST 클릭/노출/비용 → Slack #리포트 API + 크론

이미지 에셋과 검색 광고(RSA)는 별개입니다. 이미지는 향후 디스플레이/PMax 전환 시 사용하고, 지금 생성하는 건 검색 광고(SA) 텍스트 캠페인입니다.

07 지금 창업자가 해야 할 것 — Step by Step

⏰
계정 생성 타이밍 주의 Google Ads 계정 생성 순간 180만→120만 크레딧 타이머가 시작됩니다. 광고 바로 시작할 준비가 됐을 때 만드세요.
  1. Google Ads 계정 생성 ads.google.com 접속 → 신규 계정 만들기 → 업체명: 우리 나이가 어때서
    ⚠️ 이 순간 180만 크레딧 타이머 시작. 광고 준비되면 클릭.
  2. Developer Token 신청 계정 로그인 → 우측 상단 도구 및 설정 → API 센터 → 토큰 신청
    용도: "자체 계정 관리 (Self-use)" 선택 → 5~14영업일 심사
    ✅ 테스트 계정으로는 미승인 상태에서도 코드 테스트 가능     Customer ID (123-456-7890 형식) → Claude에게 전달해주세요
  3. Google Cloud Console — OAuth2 설정 console.cloud.google.com → 새 프로젝트 생성
    → API 및 서비스 → Google Ads API 활성화
    → 사용자 인증 정보 → OAuth 2.0 클라이언트 ID 만들기 (유형: 데스크탑 앱)
    → Client ID, Client Secret 복사해두기
  4. Refresh Token 발급 아래 명령어 실행하면 브라우저 창 뜸 → 구글 계정 로그인 → 토큰 발급됨 npx tsx agents/marketing/google-ads/scripts/get-refresh-token.ts
  5. 환경변수 5개 추가 — .env.local 프로젝트 루트의 .env.local 파일에 아래 추가: GOOGLE_ADS_DEVELOPER_TOKEN=여기에붙여넣기 GOOGLE_ADS_CLIENT_ID=여기에붙여넣기 GOOGLE_ADS_CLIENT_SECRET=여기에붙여넣기 GOOGLE_ADS_REFRESH_TOKEN=여기에붙여넣기 GOOGLE_ADS_CUSTOMER_ID=하이픈없는10자리숫자
