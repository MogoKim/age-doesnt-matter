Section 01

시스템 개요 — 왜 스크래퍼가 필요한가

우나어는 50·60대를 위한 커뮤니티 서비스입니다. 런칭 초기에는 실제 사용자 게시글(UGC)이 없기 때문에 콘텐츠 공백이 발생합니다. 이 문제를 해결하기 위해 두 가지 방식으로 외부 콘텐츠를 자동 수집합니다.

💡
봇:UGC 비율 관리 원칙

현재 목표 비율 95:5 (봇 콘텐츠 95%, 실제 사용자 글 5%). 서비스 성숙 후 점차 UGC 비중을 높여나가는 전략입니다.

콘텐츠 공급 전략

목표 방법 대상
실제 50·60대 UGC 수집 Track A — 네이버 카페 크롤링 우아한 갱년기, 은퇴 후 50년
인기 커뮤니티 바이럴 콘텐츠 Track B — Google Sheets 큐레이션 펨코, 오늘의유머, 네이트판
시드 콘텐츠 (AI 생성) Seed Bot — 시드봇 78명 (A~Z 26 + AA~AZ 26 + B계열 21 + EN1~EN5 5) 우나어 전 게시판

Track A ≠ Track B 목적지

Track A는 CafePost 테이블(분석/가공 대기), Track B는 Post 테이블(즉시 발행)입니다. 두 Track은 목적지 테이블이 다릅니다.

Section 04-B

화제성 파이프라인 — 즉각 HOT 달성 자동화

창업자가 화제성 탭(사는이야기_화제성 / 웃음방_화제성)에 URL을 등록하면 isFeatured=true로 저장되어, 파동(WAVE) 봇들이 자동으로 좋아요·댓글을 집중 투하해 HOT 탭에 올립니다.

🚀
2026-05-13 대폭 개선 — 글 발행 1분 이내 첫 댓글, 10분 이내 전부 완료

기존: +2분/+5분/+35분/+65분 → 변경: +1분/+3분/+6분/+10분. 파동 크론도 30분→5분으로 단축. 원본 사이트 댓글 분위기를 반영한 댓글 생성 추가.

Section 04-C · 2026-05-13 신규

원본 댓글 수집 시스템 — 분위기 반영 댓글 자동 생성

스크래핑 시 원본 사이트의 댓글을 최대 10개 수집하여 BotLog에 저장합니다. 파동 실행 시 이 댓글을 Claude에게 "분위기 참고용"으로 제공하여, 원본 반응과 비슷하면서도 50~60대 말투로 정제된 댓글을 생성합니다.

💡
왜 원본 댓글이 필요한가?

우나어 파동 댓글이 원본 커뮤니티의 반응과 전혀 다르면 글의 맥락과 어긋나 어색하게 느껴집니다. 원본 댓글의 분위기(공감/논쟁/유머 톤)를 파악하면 더 자연스러운 우리 사이트 댓글을 생성할 수 있습니다.

수집 파이프라인

1
페이지 로드 완료 직후
Playwright로 원본 사이트 페이지를 로드한 뒤, removeElements() 실행 전에 댓글 수집 시작
2
commentSelectors로 댓글 DOM 조회
사이트별 commentSelectors.item으로 개별 댓글 컨테이너 최대 10개 선택 → commentSelectors.text로 텍스트 추출
3
5자 미만 필터링
너무 짧은 리액션(ㅋㅋ, ㄱㄱ 등)은 제외. 5자 이상 댓글만 sourceComments[]에 저장
4
BotLog details에 sourceComments 포함 저장
파동 BotLog를 생성할 때 details JSON에 sourceComments 배열 포함. 나중에 scheduler가 읽어서 generator에 전달
5
파동 실행 시 Claude에게 분위기 제공
generateSheetViralComment() 호출 시 원본 댓글 최대 5개를 "분위기 참고용"으로 system prompt에 포함. 상스러운 표현은 순화, 50~60대 말투로 재작성 지시.
