A — "매일 오는 곳"이라는 루틴 소구. 커뮤니티가 이미 활성화됐다는 암시.

4 배리언트 배정 로직
핵심 원칙: 동일 기기는 항상 같은 배리언트를 봐야 함. 랜덤 배정은 리로드마다 카피가 바뀌어 측정이 오염됨 → 기기 고정 UUID 해시로 결정론적 배정.
// localStorage에 기기 고정 UUID 생성 및 저장
function getOrCreateDeviceId(): string {
  const key = 'unao_device_id'
  let id = localStorage.getItem(key)
  if (!id) {
    id = crypto.randomUUID()
    localStorage.setItem(key, id)
  }
  return id
}

function getVariant(): 'A' | 'B' | 'C' {
  // 이미 배정된 배리언트 있으면 재사용
  const cached = localStorage.getItem('signup_banner_variant')
  if (cached === 'A' || cached === 'B' || cached === 'C') return cached
  // UUID 해시 → 3등분 (결정론적)
  const deviceId = getOrCreateDeviceId()
  const hash = Array.from(deviceId).reduce((acc, c) => acc + c.charCodeAt(0), 0)
  const variant = (['A', 'B', 'C'] as const)[hash % 3]
  localStorage.setItem('signup_banner_variant', variant)
  return variant
}

12 Storage 키 전체 명세
A/B/C 테스트 신규 키
  • local unao_device_id 기기 고정 UUID. 배리언트 결정론적 배정에 사용. 삭제 시 재배정.
  • local signup_banner_variant 'A' | 'B' | 'C' — 배정된 배리언트. 기기 고정. 위너 선정 후 삭제 가능.
배너 노출 제어 키 (신규)
  • local signup_prompt_count 가입 배너 총 노출 횟수. 최대 4회 후 영구 중단
  • local signup_prompt_last_at 마지막 가입 배너 노출 시각 (ISO timestamp). 쿨다운 계산에 사용
  • local signup_prompt_done '1' — 4회 모두 노출 후 영구 중단 플래그
  • local signup_completed_at 가입 완료 시각. PWA 페이지 카운터 시작점
  • local pwa_page_views_after_signup 가입 후 콘텐츠 페이지 탐색 횟수. 3 이상 시 PWA 트리거
  • session signup_banner_dismissed '1' — 이 세션에서 가입 배너 닫은 경우. 탭 닫으면 리셋
  • session bottom_banner_shown_this_session 'signup' | 'pwa' — 하단 배너 중복 방지. 세션 내 1종만 노출
  • session signup_welcome_toast '1' — 가입 완료 환영 토스트 1회 표시용

가입 배너 노출 중 13초 PWA 타이머 도달 bottom_banner_shown_this_session 있으면 PWA 팝업 차단
이미 회원인 유저에게 가입 배너가 뜨면? useSession() 1순위 체크 → session.user.id 있으면 즉시 스킵
localStorage 전체 초기화 후 재방문 로그인 상태면 세션으로 차단. A/B 배리언트는 재배정됨 (허용 가능)
배리언트 localStorage 삭제 후 재방문 getVariant() 재실행 → 같은 deviceId가 있으면 동일 배리언트 재배정. deviceId도 삭제됐으면 새 UUID로 재배정.
A/B 테스트 중 배리언트 추가/변경 필요 시 기존 signup_banner_variant 로컬스토리지 무효화 불가 → 배포 후 신규 유저부터 적용. 기존 배정 유저는 그대로 유지.
카카오 인앱에서 카카오 로그인 제한? 우나어는 카카오 전용 → 기술적 문제 없음
가입 배너 닫고 같은 세션에서 공감 클릭 LoginPromptModal(ActionBar) 정상 노출. 별개 컴포넌트 — 충돌 없음
4회 모두 닫은 비회원이 나중에 방문 signup_prompt_done=1 → 영구 차단. 직접 로그인 클릭 시 정상 동작
20초 타이머 중 페이지 이동하면? usePathname() 변경 → 기존 타이머 clearTimeout → 새 페이지에서 20초 재시작
가입 배너와 PwaInlineBanner 동시 노출 bottom_banner_shown_this_session='signup' 있으면 PwaInlineBanner 숨김
데스크탑 가입 후 3페이지 카운터 충족 detectEnv()='desktop' → PWA 환경 아님 → PWA 이벤트 발동 안 함
신규 가입자 온보딩 중 가입 배너가 뜨면? /onboarding EXCLUDED_PATHS 포함 → 차단
A/B 실험 중 위너 배리언트 결정됐을 때 코드 정리 위너 카피 하드코딩 → getVariant() 함수 제거 → signup_banner_variant키는 GA4 분석용으로 6개월 유지 후 제거
가입 완료 후 callbackUrl이 만료된 페이지면? Next.js 404 처리. 404 → 홈(/) 이동은 기존 로직 그대로
