1 핵심 철학 & 목표 동선
"유저가 콘텐츠를 탐색하는 흐름을 끊지 않는다.
가입은 자연스럽게 유도하고, PWA는 가입 후 가치를 충분히 체감한 뒤에만 제안한다."
비회원 진입
  → 콘텐츠 탐색 (방해 없음)
  → 20초 후 하단 배너 등장 (비침습)
  → [💛 1초 가입] 클릭 → 카카오 로그인 → 온보딩
  → 가입 완료 → callbackUrl로 원래 페이지 복귀
  → 환영 토스트 표시
  → 탐색 계속 (방해 없음)
  → 콘텐츠 3페이지 탐색 후 PWA 팝업 1회
2 가입 유도 배너 — 상세 정책
노출 트리거 조건 (6개 전부 충족 필요)
  • 비회원 확인 — useSession() → session?.user?.id 없음
  • 제외 경로 아님 — SIGNUP_EXCLUDED_PATHS 미포함
  • 이 세션에서 닫은 적 없음 — sessionStorage: signup_banner_dismissed 없음
  • 총 노출 4회 미만 — localStorage: signup_prompt_count < 4
  • 쿨다운 경과 — 마지막 노출로부터 회차별 쿨다운 경과
  • 페이지 진입 후 20초 경과 — setTimeout(20,000)

인앱 브라우저 전략 — 가입 먼저 (선택지 A)
카카오/네이버/인스타 인앱에서 카카오 소셜 로그인은 기술적으로 정상 작동. 가입(핵심 KPI)을 인앱에서 먼저 달성한 뒤, PWA는 외부 브라우저 재접속 시 유도. 외부 브라우저 전환을 첫 번째 허들로 놓으면 이탈률이 높아짐.
4 PWA 유도 정책 — 가입 완료 유저 전용
트리거 조건 (6개 전부 충족)

  • 이 기기에서 가입 완료 — localStorage: signup_completed_at 있음
  • 3페이지 탐색 완료 — pwa_page_views_after_signup >= 3
  • 미설치 — getInstalled() = false
  • PWA 가능 환경 — android-chrome / ios-safari / other
  • 기존 canShow('signup') = true

6 비회원 여부 판단 우선순위
1순위 useSession() → session?.user?.id 유무 — 서버 세션 기반, 가장 신뢰
보조 localStorage: signup_completed_at — PWA 카운터 시작점으로만 사용
제외 localStorage 단독 판단 — localStorage 삭제로 우회 가능하므로 1순위 아님
핵심: localStorage를 아무리 삭제해도 세션 쿠키가 살아있는 한 로그인 유저에게 가입 배너가 절대 뜨지 않음.

/login /onboarding /signup /admin /my /terms
8 Storage 키 전체 명세
신규 추가 키
  • local signup_prompt_count 가입 배너 총 노출 횟수. 최대 4회 후 영구 중단
  • local signup_prompt_last_at 마지막 가입 배너 노출 시각 (ISO timestamp). 쿨다운 계산에 사용
  • local signup_prompt_done '1' — 4회 모두 노출 후 영구 중단 플래그
  • local signup_completed_at 가입 완료 시각. PWA 페이지 카운터 시작점
  • local pwa_page_views_after_signup 가입 후 콘텐츠 페이지 탐색 횟수. 3 이상 시 PWA 트리거
  • session signup_banner_dismissed '1' — 이 세션에서 가입 배너 닫은 경우. 탭 닫으면 리셋
  • session bottom_banner_shown_this_session 'signup' | 'pwa' — 하단 배너 중복 방지. 세션 내 1종만 노출
  • session signup_welcome_toast '1' — 가입 완료 환영 토스트 1회 표시용
기존 키 변경
키변경 내용
pwa_pending = 'signup' 즉시 실행 폐기 → signup_completed_at으로 대체
pwa_shown_this_session 유지. PWA 팝업 세션 1회 제한 그대로
9 전체 엣지케이스 처리표
엣지케이스처리 방법
가입 배너 노출 중 13초 PWA 타이머 도달 bottom_banner_shown_this_session 있으면 PWA 팝업 차단
이미 회원인 유저에게 가입 배너가 뜨면? useSession() 1순위 체크 → session.user.id 있으면 즉시 스킵. localStorage 삭제해도 세션 쿠키 유효하면 차단
localStorage 전체 초기화 후 재방문 로그인 상태면 세션으로 차단. PWA 카운터만 리셋 → 3페이지 재탐색 필요
카카오 인앱에서 카카오 로그인 제한? 우나어는 카카오 전용 → 기술적 문제 없음
카카오 인앱 가입 후 외부 브라우저 재접속 signup_completed_at localStorage 유지 → 외부 브라우저에서 페이지 카운터 재시작 → 3페이지 후 PWA
카카오 iOS URL 복사 안내 무시하는 유저 3일 쿨다운 후 재노출. 2회 무시 후 영구 숨김
PWA 이미 설치한 유저에게 가입 배너? getInstalled()=true → PWA 단계만 스킵. 가입 배너는 session.user.id로 차단
가입 후 3페이지 채우기 전 탭 닫으면? localStorage 누적 → 다음 방문에 이어서 카운트
가입 배너 닫고 같은 세션에서 공감 클릭 LoginPromptModal(ActionBar) 정상 노출. 별개 컴포넌트 — 충돌 없음
4회 모두 닫은 비회원이 나중에 방문 signup_prompt_done=1 → 영구 차단. 직접 로그인 클릭 시 정상 동작
20초 타이머 중 페이지 이동하면? usePathname() 변경 → 기존 타이머 clearTimeout → 새 페이지에서 20초 재시작
가입 배너와 PwaInlineBanner 동시 노출 bottom_banner_shown_this_session='signup' 있으면 PwaInlineBanner 숨김
데스크탑 가입 후 3페이지 카운터 충족 detectEnv()='desktop' → PWA 환경 아님 → PWA 이벤트 발동 안 함
신규 가입자 온보딩 중 가입 배너가 뜨면? /onboarding EXCLUDED_PATHS 포함 → 차단
