설계 메모
· 순서: 게시글 → 검색 1줄 → 메인 이동 줄(이전·맨앞·현재쪽 입력·이동·다음) → 근처 번호 줄(‹묶음·현재±1·›묶음) → 하단 여백.
· 펨코에서 가져옴: 검색 아래 메인 이동 줄 + 현재쪽 직접 입력·이동 + 근처 번호 + 묶음 이동.
· 우나어식 변경: 펨코 -/+ 대신 방향 화살표 ‹ ›(묶음 이동, aria-label). +는 FAB·가+와 의미 충돌, -는 삭제 오인 → 미사용. 맨뒤 버튼 없음(끝은 직접 입력). "N/총" 표시 폐기.
· 근처 번호는 현재±1 3개 중심(큰 버튼이라 억지로 5+ 노출 안 함, 모바일 최대 3개). 현재=코랄 배경. 모두 52px.
· 불가능한 액션은 숨김: 첫 페이지=이전·맨앞 미노출([현재][이동][다음]), 마지막 페이지=다음 미노출([이전][맨앞][현재][이동]). disabled 회색 버튼으로 남기지 않음 → 390px 글씨 최대에서도 1줄 유지.
· ‹=이전 묶음(현재-3), ›=다음 묶음(현재+3), 클램프 [1,총].
· 광고: 이번 미구현. 향후 조작부 아래 구분선+간격 두고 별도 PR. 앱은 라이브 웹 로드 → 웹 변경 반영, AAB 불필요.
