🏛️ 우나어 — 근본을 파헤치다 (비즈니스 프레임워크 v2)

이 문서는 빈 캔버스다. 미리 답을 적지 않는다.
창업자와 하나씩, Why를 외치며, 티키타카로 채워나간다.
각 단계가 확정되어야 다음 단계로 넘어갈 수 있다.

1️⃣ 고객의 욕망 / 니즈 / 원츠 (WHY)

"우리가 해결하려는 고객의 근본적인 문제는 무엇인가?"
⚠️ 가설 단계 (검증 필요)

데이터 소스: 카페 크롤링으로 매일 수천 개의 글이 DB에 쌓이고 있다. "어떤 주제에 좋아요/댓글이 폭발하는가"를 정량 분석하면 간접 검증이 가능하다. 추후 실제 유저 5~10명 대상 심층 인터뷰(IDI)로 직접 검증이 필요하지만, 지금은 이 데이터로도 충분히 시작할 수 있다.

채워야 할 것

  • 메인 주제 (고객이 가장 뜨겁게 반응하는 대주제)
  • 서브 주제들 (대주제 아래 세부 관심사)
  • 핵심 키워드 (고객의 입에서 반복적으로 나오는 단어들)
  • 욕망 / 니즈 / 원츠 구분 정의
⚡ 이 단계가 확정되면 → 2️⃣ 수단의 정체성, 4️⃣ 카테고리 우선순위, 5️⃣ 기능 우선순위가 자동으로 결정된다.

2️⃣ 수단: 커뮤니티 (HOW)

"그 욕망을 해결해주기 위한 우리의 무기는 '커뮤니티'다."
✅ 수단 자체는 결정됨

핵심 논점: 고객의 욕망/니즈/원츠를 해결해줄 수단은 수없이 많다. 블로그, 유튜브, 앱, 뉴스레터, 오프라인 모임... 그런데 왜 하필 커뮤니티인가? 이 질문에 본질적으로 대답해야 한다.

풀어야 할 핵심 질문

  • 인간 심리의 본질: 레딧, 디시인사이드, 펨코가 왜 거대한 기업이 되었는가? 인간의 어떤 근원적 심리가 커뮤니티를 만드는가?
  • 5060에게도 동일하게 작동하는가? 젊은 세대의 커뮤니티 욕구와 5060의 커뮤니티 욕구는 같은가, 다른가?
  • 다른 수단 대비 커뮤니티의 압도적 장점은? 블로그/유튜브/뉴스레터가 아닌 커뮤니티여야만 해결되는 것은 무엇인가?
  • 커뮤니티의 정체성: 어떤 성격의 커뮤니티인가? (이 질문의 답은 1️⃣이 확정된 후에 구체화된다)
⚡ 여기서는 구체적 기능(활력충전소, 매거진 등)을 논하지 않는다. 그것은 4️⃣ 카테고리 전략에서 다룬다.
⚡ 1️⃣이 확정되면 → 커뮤니티의 정체성과 성격이 자연스럽게 결정된다.

3️⃣ 미션 / 비전 / 철학 (WHO WE ARE)

"우리는 왜 존재하는가?"
⚠️ 암묵적으로만 존재 (명문화 필요)

채워야 할 것

  • 미션 (Mission): 우리가 매일 하는 일 한 문장. (현재: "5060이 나이 걱정 없이 일자리와 소통을 찾는 따뜻한 커뮤니티" → 너무 넓다. "일자리"와 "소통" 중 뭐가 핵심인가?)
  • 비전 (Vision): 3년 뒤 우나어는 어떤 모습인가? (없으면 OKR의 O를 세울 수 없다)
  • 철학/톤 (Tone): 우리가 고객에게 말하는 방식의 원칙 (현재 constitution.yaml에 흩어져 있음)
⚡ 1️⃣ + 2️⃣가 확정되면 자연스럽게 미션과 비전의 방향이 잡힌다.

4️⃣ 카테고리 전략 (WHAT WE OFFER)

"커뮤니티 안에서 어떤 방(카테고리)을 만들 것인가?"

현재 카테고리: 매거진 / 일자리 / 활력 충전소 / 사는 이야기

채워야 할 것

  • 이 4개가 맞는가? 빼야 할 것, 추가해야 할 것은?
  • 각 카테고리는 1️⃣의 어떤 욕망을 해결해주는가? (매핑)
  • 카테고리 간 우선순위 (MVP 카테고리는 무엇인가?)
  • 각 카테고리별 SEO 타겟 키워드
⚡ 1️⃣이 확정되면 → 카테고리 우선순위가 결정되고 → 홈의 UI/UX, 전체 UX 라이팅이 변경될 수 있다.

5️⃣ 기능(Feature) 전략

"커뮤니티에서 어떤 기능을 제공할 것인가?"

채워야 할 것

  • 현재 개발된 기능 전수 목록
  • 1️⃣ 고객 욕망 기준으로 각 기능의 존재 이유 매핑
  • 필수인데 아직 없는 기능 식별 (예: 카카오톡 공유 등)
  • 기능 우선순위 (고객 가치 × 임팩트 기준)

6️⃣ SEO 전략

"고객이 우리를 어떻게 발견하게 할 것인가?"

채워야 할 것

  • 회사 전체 SEO 전략 방향
  • 4️⃣의 각 카테고리별 타겟 키워드 맵
  • 매거진 봇 프롬프트에 SEO 키워드를 반영하는 구조
  • 유입 채널별(구글/네이버/SNS/카카오톡) 전략

7️⃣ 매출 공식 (Revenue Formula)

"돈은 어떤 구조로 벌리는가?"
월 매출 = DAU × PV/DAU × eCPM/1000 × 30일 + CPS 수익
월 순수익 = 월 매출 - 운영 비용

각 변수의 드라이버 (영향 요인) 채워야 할 것

  • DAU 드라이버: 어떤 기능들, 어떤 콘텐츠, 어떻게 유입, RR(재방문율), RR에 영향 주는 것은 또 뭔지 파고들기
  • PV/DAU 드라이버: 1인당 페이지뷰에 영향을 주는 요인들
  • eCPM 드라이버: eCPM을 높이려면 PV를 높여야 하는가? 콘텐츠 질을 높여야 하는가? 클릭 유도인가?
  • CPS 드라이버: 쿠팡 파트너스 수익에 영향을 주는 요인들
  • eCPM vs CPC 구조: 애드센스 수익 구조가 eCPM 기반인지 CPC 기반인지 정확히 파악
⚡ 이 드라이버들을 파악하면 → 어떤 부분을 건드려야 매출에 가장 큰 임팩트를 주는지 보이고 → 그것이 Initiatives가 된다.

8️⃣ OKR (Objectives & Key Results)

"이번 분기에 무엇을 달성할 것인가?"
🔴 1️⃣~7️⃣이 확정되어야 세울 수 있음

채워야 할 것

  • O (Objective): 가슴 뛰는 한 문장 목표
  • KR (Key Results): 달성을 증명하는 3~4개의 숫자 지표
  • Initiatives: 각 KR을 달성하기 위한 구체적 행동 계획 (가장 임팩트 센 것 위주)

9️⃣ 전체 흐름도 (The Big Picture)
